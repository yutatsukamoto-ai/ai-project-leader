## 1. 背景と課題認識（なぜこのSkillを作ったか）

### 既存AI生成ツールの限界（発表者Aの実体験）

- Notebook LM等の既存ツールは良いが「リアルの場で使えるスライド」が出てこないことが多い
- デザイン・レイアウトが自分のイメージ通りにならない
- 会社の固定テンプレートを入れると「ここが間違ってる」という視線が向いてしまい、かえってAI活用判断がNOになりやすい

### 従来の折衷案とその問題

> AI生成部分（図・グラフ）はスクリーンショットで切り取り、ヘッダー等はPowerPointで手作業 → 1枚なら許容できるが、スライド全体の統一感が弱くなる

### 突破口

> **GPT Image 2の登場**（現状最強クラスの画像生成モデル）。スライド1枚を1画像として生成する用途での精度が特に高い。

---

## 2. Skillのアーキテクチャ（設計思想）

### 中核の発想転換

| Before | After |
|---|---|
| 会社の固定テンプレートに合わせる | テンプレートを捨て、**ロゴだけを軸**に都度デザインシステムを生成 |
| AIが内容を要約してスライド化 | **入力テキストは100点前提**。AIは要約せず、構成最適化と割り振りのみ行う |

### インプット

```
ロゴファイル（PNG等） ＋ 内容テキスト（Markdown）
```

- 内容テキストは「読んだだけで全内容がわかる状態」が前提（要約不要・全情報入力）
- ロゴから自動でデザインシステムを生成 → それを基にスライド1枚ずつ画像生成

### アウトプットフロー

1. 最初の5枚を生成（約5分）→ 方向性確認
2. OKなら残り全枚を並列生成（サブエージェント並列化で高速化）
3. 完成後、PDFエクスポート
4. タイトル・ロゴ位置を上揃え補正（ズレ防止の後処理）

### 実績

- 47枚の勉強会スライドを**約20分**でほぼ完成品として生成
- 発表者A本人評価:「自分でゼロから作ったものを超えてきた実感」
- 現在はこのSkillのみを使用、出力をそのまま使うのが基本

## 3. Skillの設計ポリシー（品質を守るための制約）

| ポリシー | 理由 |
|---|---|
| SVGは原則使わない（GPT-4.5はSVG精度が低い） | FA等は例外的に可 |
| 1枚のスライドに全内容を詰め込まない | 人間が設計するなら分割するはず |
| 既存スライドを参照・流用させない | 古いテンプレートで手を抜くことを防ぐ |
| テキスト情報は変に要約しない | タイトル・テキストはすべて重要。省略すると意味が失われる |

## 5. Q&A ディスカッション（参加者Bとの対話）

### 参加者Bの実践アプローチ

> Markdown → Marp形式 → 社内テンプレート（背景画像当て込み）→ スライド生成

- **課題1**: 上司レビューで手戻りが繰り返し発生 → 「壊して作り直し」がしやすい形式が必要
- **課題2**: 多人数共同編集の最適フォーマット問題。「パワポ職人に最終仕上げを渡してほしい」と言われてMarkdown/PDF出力を再変換するのが苦痛

### 発展議論：PowerPoint廃止論

- 参考事例: **Mixi社がPowerPoint廃止 → 年間約10億円削減**（記事言及）
- 外部営業では最近「スライドなしでNotion投映しながら話す」スタイルも出てきている
- 発表者Aの見解: 「聞き手からすれば中身と一致しているかどうかが大事。スライドである必要自体はない」

### 議論の帰結

> スライドの目的を問い直すことが本質。人がボトルネック（謎のこだわり・頭の硬さ）になっている状態を柔らかくすることが人類の答えでは、という話になった。
