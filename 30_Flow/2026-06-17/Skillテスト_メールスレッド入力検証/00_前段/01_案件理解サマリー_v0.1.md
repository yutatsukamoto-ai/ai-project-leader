## 1. クライアント概要

- 東都リース: リース業を想定する法人向け企業
- 相談部門: 営業企画部
- 関係者: 佐伯氏（営業企画部長）、高橋氏（情報システム課長）、山根氏（法務）
- 主要システム: SharePoint、契約管理システム、ワークフローシステム、営業部門のローカルフォルダ

## 2. 相談の経緯

営業企画部から、営業の提案書作成と契約書まわりの確認に時間がかかっているため、AIで何かできないか相談があった。
メールの往復の中で、当初は「提案書作成」と「契約書確認」が混ざっていたが、法務と情報システムの補足により、初期スコープは営業向けのナレッジ検索・提案書骨子作成支援に寄っている。

現時点では、契約判断の自動化や契約書本文の横断検索は初期対象外と見るのが妥当である。
ただし、将来的な契約審査の事前確認ニーズは残っている。

## 3. 期待の構造化

| # | 期待 | 根拠となる発言・記録 | 性質 |

| W1 | 過去提案書や社内資料を探す時間を短縮したい | 「似た案件の提案書を探すだけで半日」「まず資料を探す時間の短縮」 | 業務効率化・ナレッジ検索 |

| W3 | 提案書の骨子作りを支援したい | 「過去提案や商品資料を探し、提案書の骨子作りを助ける」 | 提案作成支援 |
| W4 | 法務問い合わせの重複を減らしたい | 「営業から法務への問い合わせが重複」「FAQ化できるかもしれません」 | 問い合わせ削減・法務負荷軽減 |

W1とW3は初期テーマの中心である。
W2は効果指標になり得るが、7月前の短期デモでは測定が難しい可能性がある。
W4は法務FAQの整備として部分的に扱えるが、契約判断と混同しない線引きが必要である。
W5は初期スコープから外すべき候補である。

## 4. 制約・条件

| 項目 | 現時点の内容 | 出どころ |
|---|---|---|
| 期限 | 7月の新卒配属前に営業企画内デモを見せたい | ①クライアント指定 |
| 予算 | 未確定。役員会向けに概算費用が必要 | ②未定 |
| 期間 | 役員会向けに3か月程度の進め方が必要 | ①クライアント指定 |
| 初期対象資料 | 社内FAQ、営業マニュアル、提案書テンプレート、商品説明資料 | ①クライアント指定 |
| 初期対象外 | 契約書本文、契約判断の自動化 | ①クライアント指定 |
| データ所在 | 提案書はSharePoint、契約書は契約管理システム、稟議資料はワークフロー、古い資料はローカルフォルダ | ①クライアント指定 |
| セキュリティ | 契約書全文を外部AIに送れない。個人情報と取引条件はマスキングが必要 | ①クライアント指定 |
| 権限 | 契約管理システムは閲覧権限が細かく、権限を無視した横断検索は不可 | ①クライアント指定 |

最重要条件は、契約書・個人情報・取引条件を初期対象から外し、権限と情報管理ルールを守ることである。
営業支援の名目で契約判断や権限超え検索に踏み込むと、法務・情報システムの承認を得にくくなる。

## 5. 事実と推測の区別

**事実（入力に発言・記録あり）**

- 営業担当は過去提案書や契約条件を探すのに時間を要している。
- 提案書はSharePoint、契約書は契約管理システム、稟議資料はワークフローシステムに分かれている。
- 営業部門のローカルフォルダにも古い資料が残っている。
- 契約書全文を外部AIに送ることはできない。
- 個人情報と取引条件はマスキングが必要である。
- 法務は、契約書チェックAIのように見えることを懸念している。
- 初期対象は社内FAQ、営業マニュアル、提案書テンプレート、商品説明資料がよいという合意がある。
- 契約書本文は初期対象外である。
- 7月前のたたき台は営業企画内デモでよい。
- 成功基準は、まず資料探索時間の短縮である。

**推測（要検証）**

- 初期PoCは、営業向けナレッジ検索と提案書骨子作成支援に絞るのが適している可能性が高い。
- 法務FAQは、契約書本文検索ではなく「確認要否の案内」に限定すれば初期範囲に入れられる可能性がある。
- ローカルフォルダの古い資料は、品質と最新版管理のリスクになる可能性がある。
- 成功基準は資料探索時間の短縮を主指標にし、新人の自己解決率と法務問い合わせ削減を副指標にするのが現実的かもしれない。
