## 1. クライアント概要

- 業種: 製造業（組立型）
- 規模: 国内に複数工場を保有
- 状況: 生産管理システムを運用中だが、予定生産数と実績の乖離が常態化している

## 2. 相談の経緯

生産管理システムの予定生産数と実際の生産数が乖離していることが発覚。
原因を調査したところ、設備の稼働時間が予定より短いことが判明した。
計画外の設備停止（設備不良・操作ミス等による現場判断での停止）が報告に反映されておらず、現状の稼働時間管理＝作業員の終業時報告に依存している。
会社として初のDXプロジェクトとして発足したが、自社に前例がなく進め方が分からない状態。

## 3. 期待の構造化

| ID | 期待 | 性質 |
|---|---|---|
| W1 | 設備の稼働時間を正確に把握したい | 業務の見える化（最優先・直接的な課題解決） |
| W2 | 予期せぬ設備停止をイベントとして検知したい | 業務効率化（異常検知） |
| W3 | 設備に関する様々なデータを取得し、将来的に多様な分析ができる仕組みを作りたい | 手段先行寄り（将来ビジョン） |
| W4 | 自社のニーズや要求事項を明確にし、外部パートナーと効果的に連携できるようになりたい | 組織能力の獲得 |
| W5 | 本番導入の費用見積りの基礎情報を得たい | 予算確保（PoCの第3の役割） |

**最重要発言の特定**: 明確な不安・条件の直接発言はないが、「稼働中の既存設備に極力手を加えずに後付けで利用できるもの」という方針が繰り返し出ており、これが現場制約の核心。

## 4. 制約・条件

| 制約 | 内容 | 出どころ |
|---|---|---|
| 予算 | 上限あり（具体額は不明。デバイス1台5万円程度を目安としている） | ①クライアント指定 |
| 期間 | PoC期間は最大3カ月 | ①クライアント指定 |
| 設備への介入 | 稼働中の既存設備に手を加えない（後付け前提） | ①クライアント指定 |
| ネットワーク | 工場内LANへの新規機器接続不可（既存配線が固定レイアウト） | ①クライアント指定（現場ヒアリングで判明） |
| 本番構築 | 最終的なシステム構築は外部パートナー企業に依頼する方針 | ①クライアント指定 |
| セキュリティ | 社内に閉じたPoC。データはセンシティブでない。暗号化・認証認可は必要 | ①クライアント指定 |

## 5. 事実と推測の区別

### 事実

- 生産管理システムの予定生産数と実績が乖離している
- 原因は設備稼働時間が予定より短いこと（計画外停止が報告されていない）
- 現在の稼働時間管理は作業員の終業時報告に依存
- 設備にはPLC（プログラマブル・ロジック・コントローラ）が接続されている
- PLCにイーサネットポートが1つある
- 工場内LANは有線・固定レイアウトで新規接続不可
- フィージビリティスタディで4方式を比較し、ゲートウェイデバイス方式を第一候補に選定済み
- セルラー通信を利用する方針（工場LAN不可のため）
- 会社でキャリア回線の法人契約あり（ただし人向けプランで適さない可能性）
- 本番導入時のシステム構築は外部パートナー企業に委託予定

### 推測

- ゲートウェイデバイス方式で設備からデータ取得が可能（🟡 PoC最大の検証対象＝ノックアウトファクター）
- セルラー通信のコストが許容範囲に収まる（🟡 実機で確認が必要）
- 取得データが生産管理の予実管理に使える（🟡 生産管理部門の評価待ち）
- 工場環境（振動等）でデバイスが安定稼働する（🟡 耐環境性の検証が必要）

## 6. 不明点 → ヒアリング質問（優先度順）

1. **【最優先】** 対象設備は何台か。PoC対象は全工場か、1工場の特定ラインか
2. **【最優先】** 「予定生産数と実績の乖離」はどの程度か（数値感）。現状の業務への影響の大きさ
3. **【高】** 意思決定者は誰か。PoC結果の承認・本番導入判断は誰がするか
4. **【高】** 予算上限の具体額。デバイス・通信・クラウド含めた総予算か、デバイスのみか
5. **【高】** 外部パートナー企業は選定済みか。PoC段階から関与するか、本番導入からか
6. **【中】** 生産管理システムの種類と、将来のデータ連携時にどのようなI/Fが使えるか
7. **【中】** 工場のネットワーク環境の詳細（セルラー電波状況、設備周辺の電源確保）
8. **【中】** PoC後のパイロット運用の想定（期間・範囲）
