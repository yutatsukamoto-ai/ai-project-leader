# 30_Flow — 作業中ドラフト・試運転案件

作業中（未完成）の成果物を置く場所。完成品は `40_Stock/` へ移す。

---

## 命名規約

```
30_Flow/
└── YYYY-MM-DD/          ← 案件を開始した日付（ISO 8601）
    ├── 模擬案件_○○/     ← 模擬案件
    ├── 実案件_○○/       ← 実案件
    ├── Skillテスト_○○/  ← Skill単体の検証
    └── ○○○/             ← 案件以外の作業ドラフト
```

ルール:
- 第1階層は必ず `YYYY-MM-DD` 形式。案件名を直置きしない。
- 第2階層の接頭辞（模擬案件/実案件/Skillテスト）で種別を識別する。
- `_旧版/` は同日付内のリネーム前ファイルの退避先。
- Skill内で `{案件名}` と書く場合は、実体としては `30_Flow/YYYY-MM-DD/{案件種別}_案件名/` の案件ルートを指す。

---

## 案件フォルダの内部構造

```
模擬案件_○○/
├── project-context.md          ← 案件の共有コンテキスト（SSOT）
├── 入力_*.md                   ← Skillへの入力（議事録等）
├── _検査ログ.md                ← 自己検査の記録
├── 00_前段/                    ← フェーズ別の正典成果物
│   ├── 01_案件理解サマリー_v0.2.md
│   └── ...
├── 01_立ち上げ/
├── 02_計画/
├── 03_実行/
├── 04_監視コントロール/
├── 05_終結/
└── outputs/                    ← 中間生成物（下記参照）
```

### フェーズフォルダ vs outputs/

| | フェーズフォルダ（00_前段/ 等） | outputs/ |
|---|---|---|
| 位置づけ | **正典**。最終版の成果物 | **中間生成物**。作業途中の副産物 |
| 置くもの | 承認済み or 最新版の成果物 | サブエージェントの出力、検証レポート、旧版ドラフト |
| AIが参照すべき先 | こちらが常に優先 | 正典にない場合のみ参照 |
| 命名例 | `01_案件理解サマリー_v0.2.md` | `_verification_report_research.md`, `_research_axis1.md` |

**迷ったときの判断基準**: 「次のSkillの入力になるか？」→ Yes ならフェーズフォルダ、No なら outputs/。

AI向け優先規則:
- 正典成果物は、作成中のv0.1であってもフェーズフォルダへ置く。
- `outputs/` は正典成果物を作るための中間材料に限る。
- `project-context.md` は新規チェーン開始時に作る。古い前段停止案件で未作成の場合は、後続フェーズへ入る時点で作ればよい。
