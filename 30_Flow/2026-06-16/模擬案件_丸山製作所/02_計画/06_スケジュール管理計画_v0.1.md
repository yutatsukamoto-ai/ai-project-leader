## §1 スケジュール方針

本計画は、WBSの日程情報をスケジュールベースラインとして読み替え、遅延時に何を優先して守るかを定める。
日程データの正典は `02_計画/04_WBS_v0.1.json` とし、本書は管理方針・バッファ・進捗測定方法を定義する。

| 項目 | 方針 |
|---|---|
| 見積技法 | 段階①はボトムアップ（WBS工数ベース）、段階②③はローリングウェーブで段階①ゲート後に詳細化 |
| スケジュール単位 | 週次管理。段階①の律速作業は日単位で確認 |
| 管理ツール | wbs.json＋WBS Viewer、週次状況報告 |
| 更新頻度 | 週次。ゲート前2週間は重要タスクのみ日次確認 |
| 前提条件 | 小林様週1日対応、村上氏ヒアリング参加、藤田様のデータ抽出協力 |

## §5 バッファ管理方針

| 項目 | 方針 |
|---|---|
| プロジェクトバッファ | ①ゲート前に5営業日相当を確保。1.2/1.3の遅延吸収に使う |
| フィーディングバッファ | 藤田様の基幹データ抽出に3営業日。ISO確認に2営業日 |
| 消費ルール | バッファ1/3消費で注意、2/3消費でリカバリープラン、全消費見込みで変更管理表起票 |
| 報告 | 週次状況報告のスケジュール欄にバッファ消費を記載 |
| 根拠 | R-A（村上氏協力拒否）とR-C（大西氏不在）の影響を反映 |

## §6 進捗測定方法

| 項目 | 内容 |
|---|---|
| 測定タイミング | 週次。1.2と1.3は週2回の短い確認 |
| 測定方法 | マイルストーン達成率、WBSタスク完了率、バッファ消費率 |
| 遅延の判断基準 | クリティカルパス上のタスクで計画日から3営業日以上遅延、またはバッファ2/3消費 |
| 遅延時の対応 | PMが小林様に即時共有し、必要に応じて大西様へエスカレーション |
| ツール | WBS Viewerのイナズマ線、週次状況報告、変更管理表 |
