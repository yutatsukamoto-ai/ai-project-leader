## §1 スコープ管理方針

本計画は、丸山製作所 受注〜出荷プロセス改善における「やること・やらないこと」の境界を固定する。
統合計画書§2はスコープの骨格、本書は要求事項一覧・スコープ定義書・WBS辞書の粒度で境界を管理する。

| 項目 | 方針 |
|---|---|
| ベースライン | 本書v0.1承認時点のIn-Scope / Out-of-Scope / WBS辞書をスコープベースラインとする |
| 変更管理プロセス | スコープ変更は変更管理表に起票し、影響評価（リードタイム・予算・関係者負荷）を行ったうえで大西様または委任範囲内の小林様が承認する |
| 受入基準の定義方法 | 各要求事項・WBS辞書に受入基準を置き、段階ゲートで現場・業務改善推進室・意思決定者が確認する |
| スコープ検証の頻度 | 段階①終了、段階②終了、段階③中間、終結時に実施する |
| 未確定の扱い | 🟡項目は確定タイミングを明記し、未確定のまま対外約束にしない |

## §3 スコープ定義書

### 含むもの（In-Scope）

| ID | 含むもの | 成果物 / 受入基準 |
|---|---|---|
| IN-01 | 受注〜出荷プロセスの現状把握 | ヒアリング記録、As-Is業務フロー、基幹データ分析 |
| IN-02 | ボトルネックの定量化 | 工程別所要日数・ミス件数を含む分析レポート |
| IN-03 | 改善策の設計 | To-Be業務フロー、施策比較表、優先順位提案 |
| IN-04 | クイックウィン実行 | 最優先施策の実行記録、効果測定レポート |
| IN-05 | 段階ゲート運営 | ①→②、②→③、③中間の判断材料作成 |

### 含まないもの（Out-of-Scope）

| ID | 含まないもの | 理由 / 必要時の扱い |
|---|---|---|
| OUT-01 | 基幹システム（オービック7）の入替・後継選定 | スコープ混同を防ぐ。必要なら別プロジェクト化 |
| OUT-02 | 全業務の一括BPR | 受注〜出荷に限定。全社展開は成果確認後 |
| OUT-03 | 製造工程の自動化・設備投資 | 本案件は業務フロー改善。設備投資は別途稟議 |
| OUT-04 | ●●工業契約更新の保証 | 外部制約であり、本案件は判断材料と改善施策を提供する |

### 前提条件・制約条件

| 区分 | 内容 | 崩れた場合 |
|---|---|---|
| 前提 | 小林様が週1日対応可能 | 頻度を下げ、当社側の準備を厚くする |
| 前提 | 村上氏が段階①に参画する | R-Aを発動し、小林様＋パートで代替。ただし品質低下を明示 |
| 制約 | 予算1,500万円以内 | コスト管理計画と変更管理表で調整 |
| 制約 | 専任なし・兼任前提 | 資源管理計画で負荷上限を管理 |

## §5 スコープ検証の方法

| タイミング | 検証方法 | 検証者 | 不合格時の対応 |
|---|---|---|---|
| 段階①ゲート | As-Isフロー・分析レポートのレビュー | 大西様・小林様・村上氏 | ②へ進まず、追加ヒアリングまたは分析を行う |
| 段階②ゲート | To-Be案・施策比較表のレビュー | 大西様・山田工場長・小林様 | 施策の絞り込みをやり直す |
| 段階③中間 | 効果測定と旧運用切替条件の確認 | 大西様・現場責任者 | 旧運用廃止を延期し、変更管理表へ起票 |
| 終結時 | 成果要求R-01〜R-03の達成度確認 | スポンサー・PM | 教訓化し、未達は次フェーズ課題へ申し送る |
