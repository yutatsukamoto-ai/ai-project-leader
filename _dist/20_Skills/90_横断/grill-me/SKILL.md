# grill-me — 1問ずつ詰めて共通理解に到達する横断の思考道具

フェーズを跨いで使える思考道具。
設計・計画・決定の「依存ツリー」を、推奨回答つきの質問で1つずつ潰し、認識のずれがゼロになるまで詰める。
成果物を作るSkillではなく、それらを支える道具（道具を作る道具）。
出典＝Matt Pocock の grill-me（"Interview me relentlessly… resolving dependencies one-by-one"）を、本プロジェクトの協働スタイルに合わせて再構成した。

## いつ使うか（適用範囲・テーラリング）

- **明示呼び出し** … 「grill-meして」「これ詰めたい」「壁打ちで詰めて」等。
- **AI能動サジェスト** … 重い／不可逆／曖昧な分岐を検知したら「ここ詰めますか？」と一声かける。発動するかは本人がGO。
- **主な対象** … 前段(案件理解0-1／課題仮説0-2)の曖昧さ潰し／自分の保留中の決定を詰める。
- **全部には当てない** … 要る場面だけ。当てすぎ＝540問化を避ける（テーラリング）。
- 調べれば分かること（コード・既存ファイル・資料・Web）は質問せず、自分で確認してから聞く。

## やり方（背骨）

1. **言い換えから始める** … 依頼を自分の言葉で言い換え「この理解で合っていますか？」を冒頭に置く。明確に見える依頼ほど暗黙の前提がずれている。
2. **依存ツリーを描く** … 何が何に依存するかを並べ、上流の依存から潰す。順番を勝手に飛ばさない。
3. **1問ずつ＋推奨を添える** … 質問は1問ずつ。AskUserQuestion を使い、推奨回答を先頭に「(推奨)」と理由つきで出す。一度に複数論点を投げない。
4. **言語化し直して返す** … 答えを自分の言葉で返し、含意（本人がまだ言語化していない帰結）を提示する。Whyから逸れていれば理由つきで率直に指摘する（歓迎されている）。
5. **ずれがゼロになるまで繰り返す** … 依存が出尽くす／認識が揃うまでループ。飛ばす論点は「なぜ飛ばしてよいか」が妥当になってから飛ばす。
6. **可逆は攻める・不可逆は関所** … 不可逆な分岐ほど丁寧に詰める。可逆な分岐は仮決めで前に進めてよい。

## 残し方（出力）

grill-me 自身は固定の出力ファイルを持たない。
最後に「結論ブロック」だけを出す：

- **決まったこと**
- **前提・根拠**
- **残る論点**
- **推奨次アクション**

残し先は文脈に従い、既存の正典へ流す（grill-me専用ログは作らない＝二重化しない）：

- **前段で使った** → 各成果物（案件理解サマリー／課題仮説シート）の中身へ流す。
- **自分の決定** → 懸念マスター（解決ログ）＋決定の索引＋メモリーへ流す。決定の残し方の既存ルールに従う。

## してはいけないこと

- 一度に複数論点を投げる／推奨なしで丸投げする。
- 全部に当てて540問化する。
- 調べれば分かることを質問する。
- 結論を勝手に確定する／勝手にファイル化する（判断は本人・残し先は既存正典）。
- grill-me専用ログを作って懸念マスターと二重化する。
