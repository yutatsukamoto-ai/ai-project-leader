# 90_横断（横断Skillの枠）

`20_Skills` 配下のフェーズ別Skill（00_前段〜05_終結）に対して、**フェーズをまたいで効く"Skill"**を置く枠。

- ここに置くのは **フェーズを跨いで使うSkill束**。実行を直接支える「実行支援Skill」だけでなく、**思考道具（詰め・壁打ち・レビュー等の進め方の道具）**も含む。
  - 例: `grill-me`（詰めの思考道具）、将来の文章レビューSkill・作図補助Skillなど。
- **横断ガイドラインの"ドキュメント"はここに置かない**。それらの正典は `40_Stock/横断ガイドライン/` にある（文章の書き方＝`文章作法.md` など）。
- **メタSkill（Skillやプロジェクト自体を作る／立ち上げる道具）はここに置かない**。それは `99_メタ/`（例: `skill-kabeuchi-kickoff`・`_seikabutsu-template`）。
  - 住み分けの目安: ここ90_横断＝**進行中の作業の中で使う働く道具**／99_メタ＝**システム自体を作る・立ち上げる道具**。

意味の混同（Skill束とドキュメント置き場、横断とメタ）を避けるための区分。

## 定義拡張の記録（2026-06-15）

当初の定義は「フェーズ跨ぎの**実行Skill**用」だった。第1号候補 `grill-me`（成果物を作らない思考道具）の設置にあたり、定義を **「実行支援・思考道具を含む、フェーズを跨いで使うSkill」** へ広げた。
- **空枠だった理由**: フェーズを跨ぐSkillが生まれるまでの予約。空＝異常ではない。
- **適用条件**: ①フェーズを跨いで使う ②成果物（フェーズの基幹デリバラブル）そのものではない ③メタ（システム/プロジェクトを作る道具）ではない。三つを満たすものを置く。

> この拡張は成果物マップ「横断・メタ」表／決定の索引「横断の住み分け(M-12)」と整合させてある。
