# PMBOK第8版 6原則一覧

【この文書について】
PMBOK® Guide 第8版（2025年11月 PMI発行）の6つのプロジェクトマネジメント原則と、
新設の「プロジェクトマネジメント・マインドセット」の整理。
40プロセスが「何をやるか」なら、6原則は全Skill共通の「どう判断するか」の軸。
40プロセス一覧（PMBOK第8版_40プロセス一覧.md）とセットで使う。

【注意】
日本語訳は参考訳（第8版日本語版の公式訳が未確認のため）。
作成日: 2026-06-12

---

## 6原則

| # | 原則（英語） | 参考訳 | 要点 |
|---|---|---|---|
| 1 | Adopt a Holistic View | 全体俯瞰の視点を持つ | システム思考。構成要素と相互依存を理解する。第7版の「システム相互作用」＋「複雑さへの対処」を統合 |
| 2 | Focus on Value | 価値に集中する | 価値こそ成功の最終基準。適応性・回復力・変革実現の概念も吸収 |
| 3 | Embed Quality Into Processes and Deliverables | プロセスと成果物に品質を組み込む | 旧「品質を作り込む」を精緻化。サステナビリティ・コンプライアンスの観点を追加 |
| 4 | Be an Accountable Leader | 説明責任を果たすリーダーであれ | スチュワードシップ＋リーダーシップ行動を統合。誠実さ・自己認識・共有リーダーシップ |
| 5 | Integrate Sustainability Within All Project Areas | あらゆる領域にサステナビリティを統合する | 第8版で完全新設。環境・社会・経済のトリプルボトムライン。回避→最小化→補償のピラミッド |
| 6 | Build an Empowered Culture | エンパワーされた文化を築く | 旧「協働的なチーム環境」から発展。チームだけでなく全ステークホルダーに拡大 |

---

## マインドセット枠組み（第8版で新設）

6原則を3つの志向で束ねる枠組み。第7版には存在しない。

| 志向 | 参考訳 | 対応する原則 |
|---|---|---|
| Proactive | 先取り | #1 全体俯瞰 ＋ #3 品質の組み込み |
| Ownership | 当事者意識 | #4 説明責任あるリーダー ＋ #6 エンパワーされた文化 |
| Value-Driven | 価値駆動 | #2 価値への集中 ＋ #5 サステナビリティ統合 |

## 第7版の12原則からの統合マッピング

| 第7版の原則 | 第8版での行方 |
|---|---|
| 勤勉で敬意を払い面倒見の良いスチュワードであること | → #4 Be an Accountable Leader に統合 |
| 協働的なプロジェクトチーム環境を構築する | → #6 Build an Empowered Culture に発展 |
| ステークホルダーと効果的に関わる | → #2 Focus on Value ＋ ステークホルダードメインへ |
| 価値に焦点を当てる | → #2 Focus on Value として維持 |
| システムの相互作用を認識・評価・対応する | → #1 Adopt a Holistic View に統合 |
| リーダーシップを示す | → #4 Be an Accountable Leader に統合 |
| 状況に基づいてテーラリングする | → ガイド第3章（テーラリング）へ移動（原則からは外れた） |
| プロセスと成果物に品質を組み込む | → #3 として維持・精緻化 |
| 複雑さに対処する | → #1 Adopt a Holistic View に統合 |
| リスク対応を最適化する | → リスクドメインへ移動（原則からは外れた） |
| 適応力と回復力を持つ | → #2 Focus on Value に統合 |
| 想定した将来状態の達成に向けて変革できるようにする | → #2 Focus on Value に統合 |

## Skillチェーン設計での使い方

各Skillの判断基準・レビュー観点を書くとき、この6原則を共通の物差しにする。
例: 成果物レビューSkillに「#2 価値への集中の観点で、この成果物は誰のどんな価値に繋がるか」を組み込む。
マインドセット3志向（先取り・当事者意識・価値駆動）は、Skillのトーン設計（指示の温度感）に使える。
