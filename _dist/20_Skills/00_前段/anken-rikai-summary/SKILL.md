# 案件理解サマリー（前段チェーン 0-1）

## 目的

クライアントの曖昧な案件説明を「検証可能な理解」に構造化し、✋理解確認を通すこと。
このSkillの本体は文書生成ではなく、**停止ポイントでの理解のすり合わせ**にある。

## 入力

- 案件説明: ①口頭テキスト ②議事録ファイル ③スライド資料（PDF/PPTX）
- ③の場合のみ: 案件理解を抽出 → 抽出結果を提示して✋認識確認 → 承認後に手順2へ

## 手順

1. 入力を読む。`references/型と品質基準.md` を読む
2. v0.1を作成（型・判断基準はreferences準拠。文体は `references/文章作法.md` 準拠）
3. ✋ **停止【スキップ禁止】**: 「この理解で合っていますか？」+ 優先度順の質問リストを提示
4. 回答を信頼度記号（【談】【未】）付きで反映し、v0.2を作成。「回答者が知らないこと」自体を重要な発見として明記
5. ユーザー承認後、次工程（0-2 課題仮説シート）へ受け渡し

## 停止ポイント

- 手順3の理解確認は必須。承認なしに0-2へ進まない（0-1の誤りは以降すべてに伝播するため）

## 出力

- `00_前段/01_案件理解サマリー_v0.1.md` → 確認反映後 `v0.2.md`（修正のたびに版を上げる）
- 各バージョンの冒頭に「入力: ○○」を明記する

## ルール

- 入力は案件説明と本referencesのみ。前提を勝手に補わない（不明点は質問リストに回す）
- 多段階を勝手に進めない。停止ポイントごとにユーザーの判断を待つ
- **承認後、確定した新事実と承認イベントを `project-context.md` の「確定情報／決定事項ログ」に書き戻す**（確定のみ。推測は本成果物内に留める）。後工程がcontextだけで前提を把握できる状態にする
