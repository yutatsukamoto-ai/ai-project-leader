# 案件教訓（脱識別済みの振り返り集）

役割: 終わった案件の「振り返り＝倒した敵の記録」を貯め、**次に似た案件が来たら引いて再戦できる**ようにするStock。
位置づけ: 配布の3層モデルのL2寄り（横断で再利用する知識）。D-01決着（2026-06-15）に基づく。

## 二段の住み分け（重要）

| 版 | 中身 | 置き場 | 配布 |
|---|---|---|---|
| 固有版 | 社名・人名・金額込みの生の振り返り | 各案件の `05_終結/`（終結報告書） | ✕ 内部限定・配布除外 |
| **汎用版（ここ）** | 脱識別した教訓（業種・規模は一般化） | `40_Stock/案件教訓/` | ◎ 配布可 |

固有版（倒した敵の素顔）は案件フォルダに残す。ここに置くのは、**社名・金額を伏せて誰の案件か分からなくした教訓**だけ。

## 入れるときの手順（終結時）

1. 案件の `05_終結/` の終結報告書（固有版）を要約する。
2. **脱識別**: 社名→「ある製造業の中堅企業」等／金額→レンジor削除／個人名→役割名。
3. `_案件教訓テンプレ.md` をコピーして1枚書く。ファイル名は内容で（例: `製造業_施工計画AI_PoC立上げ.md`。社名は使わない）。
4. Stockへ出す前に脱識別チェック（配布§6の①〜⑤）を指差し確認。

> 関連: `00_プロジェクト管理/配布と雛形化_層の定義と棚卸し.md` §6（脱識別チェック）／接続規約 決定3。
