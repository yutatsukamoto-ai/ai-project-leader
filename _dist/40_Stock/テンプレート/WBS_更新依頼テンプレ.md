# WBS 更新依頼テンプレ（正典）

目的: 作業者が「進捗」をこの枠で報告 → AI（更新係）が `wbs.json` に正確に反映する。
作業者は**ツールを触らなくてよい**。**該当する行だけ**書けばよい。
※これは空欄の正典。案件で使うときはコピーして記入し、案件フォルダに置く。

---

## 記入欄（これをコピーして埋める）

```
■WBS更新依頼
案件名: 〔            〕
報告者: 〔      〕   報告日: 〔YYYY-MM-DD〕
――― 更新したい行だけ、下のタグで1行ずつ ―――
[着手]     No.〔  〕〔作業名〕 開始日〔YYYY-MM-DD〕
[完了]     No.〔  〕〔作業名〕 終了日〔YYYY-MM-DD〕
[予定変更]  No.〔  〕〔作業名〕 予定開始〔  〕 / 予定終了〔  〕
[担当変更]  No.〔  〕〔作業名〕 新担当〔      〕
[追加]     親No.〔  〕の下に 〔作業名〕 予定〔開始〕〜〔終了〕 担当〔  〕 工数〔  人日〕
[補足]     No.〔  〕〔自由に一言〕
```

## 記入ルール（誤解を防ぐ3つだけ）

1. **No.は必須**（閲覧版の左端の番号）。取り違え防止に〔作業名〕も併記する。
2. **日付は `YYYY-MM-DD`**（`7/17` でもOK・AIが直す）。
3. **該当する行だけ**書く。使わないタグの行は消してよい。

## AIが何をするか（タグ → 反映先を固定）

| タグ | `wbs.json` での反映 |
|---|---|
| [着手] | そのNo.の `actual.start` に日付 |
| [完了] | `actual.end` に日付（行がグレー＋✓・イナズマ線に反映） |
| [予定変更] | `plan.start` / `plan.end` を変更 |
| [担当変更] | `assignee` を変更 |
| [追加] | 親No.の `children` にリーフ追加（id自動採番・予定/担当/工数を反映） |
| [補足] | `note` に追記 |

反映後、AIは**変更点を一覧で要約して返す**。人が✋確認してから確定する（誤反映を防ぐ関所）。

## 記入例

```
■WBS更新依頼
案件名: 山藤商事
報告者: 佐藤   報告日: 2026-07-17
[完了] No.1.1 現場ヒアリング     終了日 2026-07-17
[着手] No.1.4 対象業務の選定     開始日 2026-07-28
[補足] No.1.3 失敗要因調査、日程調整で2日遅れ見込み
```

→ AIの返し（例）: 「1.1を完了(7/17)、1.4を着手(7/28)に更新、1.3のnoteに遅延見込みを追記しました。反映してよいですか？」
