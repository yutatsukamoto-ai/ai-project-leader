# カード（ドメインナレッジの差し込み層）

成果物Skillの本体を薄く保ったまま、必要なときだけ引く「型・勘所・処方」の小片。
本線（Skill・骨子）を太らせないため、ここに分離する。正典はこのREADME。

## 2型と住み分け（最重要）

カードは一種類ではない。**効き先で2型に分ける**（出典思想＝源泉2型の住み分け）。

| 型 | 置き場 | 効き先 | 形 | 例 |
|---|---|---|---|---|
| **レシピ型** | `レシピ_作成/` | 作成・品質（②差し込み） | 良い作り方の勘所 | スコープ定義/WBSの作り方、ステークホルダー登録簿の作り方 |
| **症状型** | `症状_源泉/` | 監視（遭難本・関所） | 症状→処方 | チーム動かない＝学習性無力感、数字だけ追う＝グッドハートの法則 |

- レシピ型 → 各成果物Skillの手順「中身を埋める」の**差し込み枠**から引く。
- 症状型 → 監視が症状を検知したら、その**症状タグで引いて処方を差し出す**（能動サジェスト）。
- **無理に繋がない**。レシピ型を監視に、症状型を作成に流用しない（効き先タグで弾く）。

## 共通スキーマ（frontmatter）

```yaml
---
name: <短いスラッグ>
型: レシピ | 症状
効き先: 作成 | 監視 | 両方        # この1語で振り分ける
対象: <フェーズ・成果物>            # レシピ型。例「立ち上げ/ステークホルダー登録簿」
症状トリガー: <症状の言い回し>        # 症状型のみ。引くときの索引
出典: <内部参照ポインタ>            # 書名・章。配布時はアイデア再構成のみ
最終更新日: YYYY-MM-DD
配布可否: 可 | 内部のみ              # 著作物・実データ起因は「内部のみ」
---
```

本文の型:

- **レシピ型** … 勘所（3〜5点）／作る順番／よくある罠／良い例の方向（転載しない）。
- **症状型** … 症状（トリガー）→ 一言の処方 → 効きやすいフェーズ → 効き先（どの判断を変えるか）。

## ルール（死蔵させない番人）

- 入れる基準は**「トリガーが明確 ＆ 判断を変える」だけ**。面白いだけは入れない。
- 1カード＝1ファイル。薄く。出典は内部参照OK・配布版はアイデア再構成（著作物・出典明記・図版転載はしない）。
- 定期棚卸しで陳腐化を間引く（最終更新日で判定）。
- 記号は横断ガイドラインの正典に従う（確度/採否/状態を流用しない）。
