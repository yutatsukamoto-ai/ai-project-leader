### 境界・運用メモ

- **出力フォーマット規約と文章作法§Eの線引き**: 配色の大原則は規約、Word固有の体裁細目（フォント・見出し色＋下線・絵文字置換）は §E。重複させない。§Eを規約へ統合する案は保留。
- **図の書き方**: `図の書き方.md`に独立済（②C節からリンク）。背骨＝ユーザー提供の図解カタログ（定性/定量2系統）、穴埋め＝書籍第5章。色は②B節、単位・凡例は表と数値、へ接続。
- **記号の正典**: 確度（●◐○）・テーラリング採否（◎○△）・未確定（🟡）の定義は `表と数値の見せ方.md` が正典。同じ記号を別の軸（状態・進捗等）に流用しない。既存の断片（課題仮説シート＝確度、計画提案書＝採否、文章作法§D＝未確定）は今後この正典に合わせる。
- **同期**: 出力フォーマット規約・表と数値の見せ方は現時点で実行時に読むSkillが無いため sync 対象外。参照するSkillが出たら sync-manifest に追加する。

## SSOT規約（重要）

`文章作法.md` の**正典はこのファイル**。編集は必ずここで行う。

各Skillの `references/文章作法.md` は、配布（`.skill`の自己完結）のために置く**同期コピー**であり、直接編集しない。`.skill`はzipで固められて配布・インストールされ、実行時は自分のバンドル内しか読めないため、コピーの物理的存在は必須（消せない）。だからこそ「どれが原本か」を1つに固定する。

### 同期先（5 Skill）

- `20_Skills/00_前段/anken-rikai-summary/references/文章作法.md`
- `20_Skills/00_前段/kadai-kasetsu-sheet/references/文章作法.md`
- `20_Skills/00_前段/research-memo/references/文章作法.md`
- `20_Skills/00_前段/keikaku-teiansho/references/文章作法.md`
- `20_Skills/01_立ち上げ/project-charter/references/文章作法.md`

### 更新手順

1. このフォルダの `文章作法.md`（正典）を編集する。
2. `bash _tools/build.sh --sync` を実行 → 全コピーへ反映＋該当`.skill`を再パッケージ＋verify（対象は`_tools/sync-manifest.tsv`で管理）。
3. 再パッケージされたSkillは Save skill カードから再登録する（[[skill-update-one-click-flow]]）。

> ズレ確認だけなら `bash _tools/build.sh --check`（非破壊）。`build.sh` が同期・検査・パッケージ化・点検の唯一の入口。

> コピーは正典をそのまま写す（ヘッダ等は足さない）。正典＝コピーがバイト一致である状態を保つ。
