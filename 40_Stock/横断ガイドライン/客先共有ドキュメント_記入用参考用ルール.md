# 客先共有ドキュメント：記入用・参考用 2ファイルルール

> 適用範囲: お客様に渡して記入してもらうすべてのドキュメント（ヒアリングシート・アンケート等）

---

## ルール

お客様が記入するドキュメントは、**必ず2ファイルで作成する**。

| ファイル | 用途 | セルの背景 | 記入例テキスト |
|---------|------|-----------|--------------|
| 記入用 | お客様に渡す。ここに記入してもらう | **グレー**（「ここに書く」の合図） | なし・空欄 |
| 参考用 | お客様が横に置いて参照する | **白**（記入しないので色の区別不要） | 通常の黒テキストで表示 |

---

## なぜこのルールが必要か

ヒントテキストを記入欄に直接書くと、次の問題が発生する。

1. **消す手間** — 記入前に例示テキストを削除しなければならない
2. **参照できない** — 消したら例を見られなくなる

2ファイル方式なら、参考用を見ながら記入用に書けるため両方が解決する。

---

## 命名規則

```
{ファイル名}_記入用.docx   ← お客様渡し用
{ファイル名}_参考用.docx   ← お客様が参照する用
```

例:
- `00_ヒアリングシート_PoC_記入用.docx`
- `00_ヒアリングシート_PoC_参考用.docx`

---

## 格納場所

どちらも同じ案件フォルダに並べる。

```
30_Flow/実案件/{案件フォルダ}/00_前段/
  ├── 00_ヒアリングシート_PoC_記入用.docx   ← お客様に渡す
  └── 00_ヒアリングシート_PoC_参考用.docx   ← 横に置いて参照
```

## docx生成スクリプトでの実装パターン

```javascript
// mode で分岐: 'kinyuu' | 'sanko'
function makeDoc(mode) {
  function fc(hint, w) {
    if (mode === 'sanko') {
      // 参考用: 全セル白背景・通常テキスト（ヒントなし空セルも白）
      return new TableCell({
        shading: { fill: "FFFFFF" }, ...
        children: [new Paragraph({ children: [new TextRun({ text: hint || "　", size: 18 })] })]
      });
    }
    // 記入用: グレー背景・空欄
    return new TableCell({
      shading: { fill: "F5F5F5" }, ...
      children: [new Paragraph({ children: [new TextRun({ text: "　", size: 18 })] })]
    });
  }
  // ... 本文（共通）
}

// 2ファイル同時生成
Promise.all([
  Packer.toBuffer(makeDoc('kinyuu')).then(buf => fs.writeFileSync("記入用.docx", buf)),
  Packer.toBuffer(makeDoc('sanko' )).then(buf => fs.writeFileSync("参考用.docx", buf)),
]);
```
