# 図の書き方（全Skill共通）

「何を伝えたいときに、どの図を使うか」の正典(SSOT)。図の選び方と共通の作法だけを薄く担う。
各図の細かい描画手順（きれいなツリーの組み方等）は入れない＝選び方と原則に絞る。
背骨: 図解の型カタログ（`10_参考資料/図・グラフ関連 参考画像/` の図表4-11「定性チャート」・4-12「定量チャート」。出典書名は未確認・要点を再構成、図版は転載しない）。
穴埋め: ビジネスダッシュボード本 第5章（定量の選択ロジック・ノイズ削減・視線）。
接続: 色は `出力フォーマット規約.md` B節（3色）、単位・凡例・確度記号は `表と数値の見せ方.md`、1図1メッセージは文章作法の見出し思想。
確定経緯: 横断ガイドライン候補①として2026-06-13に着手（②C節の予約席を本ファイルへ独立）。

## A. まず2系統に分ける

図は2種類。どちらを描くのかを先に決める。

- **定性図** — 考え・関係・構造を示す（数値が主役でない）。例: 課題の分解、業務の流れ、ポジショニング。
- **定量グラフ** — 数値そのものを示す。例: 売上推移、構成比、項目比較。

迷ったら問いは1つ。**伝えたいのは「考えの構造」か「数値の大小・変化」か。**

## B. 定性図の選び方（やりたいこと → 型）

よく使う中核を主役に。高度な型は「その他・必要時」に置く（シンプル優先）。

**中核（まずここから選ぶ）**

| やりたいこと | 型 |
|---|---|
| 分解する（課題・施策・KGI/KPIを枝に） | ツリー |
| 評価・比較する（2軸で位置づけ／候補を選ぶ） | マトリクス、評価テーブル |
| 流れを示す（工程・手順・業務） | フロー図、ステップ図 |
| 時間で並べる（計画・道のり） | ロードマップ |
| 階層・構造を示す（組織・業界・積み重ね） | ピラミッド、レイヤー |

**その他（必要なときだけ）**

要素間の影響関係を示すインフルエンスダイアグラム、事業の全体像を示すビジネスモデル図、項目の散らばりを示す分布図。読み手に説明が要る型なので、中核で足りるなら使わない。

## C. 定量グラフの選び方（比較したいこと → グラフ）

選択は3つの掛け合わせで決める（書籍の選択ロジック）。
**比較方法（大きさ／変化／構成比／分布／関係性）× 比較軸の数 × 時間軸の有無。**

**中核（まずここから選ぶ）**

| 比較したいこと | グラフ |
|---|---|
| 変化・推移（時間軸あり） | 折れ線グラフ |
| 大きさ・項目の比較 | 縦棒／横棒グラフ（項目名が長い・数が多いなら横棒） |
| 構成比（時間で並べる） | 積み上げ棒グラフ |
| 分布（1変数の散らばり） | ヒストグラム |
| 2変数の関係性 | 散布図 |

**その他（必要なときだけ）**

3つ目の量を円の大きさで足すバブルチャート、構成比と量を同時に見せるモザイク／パーメッコ、増減の内訳を見せるウォーターフォール。

**棒グラフは万能。** 大きさ・構成比・ランキングまで広く使える。迷ったらまず棒。
**数表・クロス集計は最小限。** 読む負荷が高い。重要なものだけにし、大小比較は棒グラフで代替する（→ `表と数値の見せ方.md`）。

## D. 共通の作法（どの図にも効く）

- **目的と利用者を先に決める** — 「何のために作るか」「誰が見るか」を決めてから描く。これが曖昧だと、複雑化して誰も見ない図ができる。
- **粗から細へ** — 全体の概要を先に描き、必要なところだけ詳細化する。いきなり細部から描くと挫折しやすい。
- **シンプル優先** — 読み方の説明が要る特殊な図は避ける。基本の型で分析・説明が足りるなら、凝った図は使わない。
- **ノイズを削る（シグナル/ノイズ）** — 読み取りに不可欠な要素（シグナル）を残し、余計な罫線・背景・影・3D・装飾色（ノイズ）を削る。ただし色が意味を持つとき（積み上げの区分など）はシグナルなので残す。
- **視線の流れ** — 重要な図ほど左上・中央へ。1画面はZ型、縦長はF型。視線の順番と説明の順番をそろえる。
- **色は3色ルールに従う** — メイン基調、強調はアクセントを数%だけ（→ `出力フォーマット規約.md` B節）。系列の色を増やしすぎない。
- **単位・凡例・確度** — 単位明示・凡例の出し方・確度記号は `表と数値の見せ方.md` に従う。
- **1図1メッセージ** — 図のタイトルに「何が言いたいか」を書く（「月別売上」より「売上はQ2に頭打ち」）。図と結論を一致させる。
- **更新を想定して作る** — 一度きりの図か、継続的に更新する図かを意識する。更新する図は管理者・更新タイミング・最新版の置き場をセットで決めておく。決めないと形骸化する。
