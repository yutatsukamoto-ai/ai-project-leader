# 用語集（横断ガイドライン④）

成果物・Skill・規約で使う語の定義を1枚に固定する正典(SSOT)。語のブレ（ドリフト）を防ぐためのもの。
記号の定義は `表と数値の見せ方.md` が正典。本書はそれを含む語彙全体の索引＝重複する記号定義はここでは要約に留め、正典へリンクする。

---

## 1. フェーズ・工程の語

| 語 | 定義 |
|---|---|
| フェーズ（局面） | PMBOKの5局面（立ち上げ／計画／実行／監視・コントロール／終結）＋本プロジェクト独自の前段。`20_Skills`・案件フォルダのサブフォルダ名と一致。 |
| 段階 | クライアント案件の実行ロードマップの区切り（例: 段階①②③）。**PMBOK局面とは別軸**。混同しない。 |
| 前段 | 案件の入口（0-1案件理解〜0-4計画提案）。承認後に本体（立ち上げ以降）へ引き継ぐ。 |
| 本体 | 立ち上げ以降のPMBOK局面。前段の成果を引き継ぎ書で受け取る。 |
| #番号 | PMBOK第8版40プロセスの通し番号。正典＝`10_参考資料/PMBOK第8版_40プロセス一覧.md`。プロセス参照は必ずこの#番号で行う。 |

## 2. 状態を表す語（記号に流用しない）

状態は**言葉**で書く。確度記号（●◐○）・採否記号（◎○△）を状態の代わりに使わない（`表と数値の見せ方.md` A節）。

| 文脈 | 使う語 |
|---|---|
| 状況報告の領域状態 | 順調／注意／要対応 |
| 終結報告書の達成度 | 達成／一部／未達 |
| 課題管理表の状態 | オープン／対応中／完了 |
| 変更管理表の状態 | 起票／評価中／承認待ち／承認／却下／反映済 |
| リスク登録簿の状態 | オープン／対応へ移管／顕在化済 |
| 確定度（チャーター等） | ✅確定／🟡未確定／➖対象外 |

## 3. 記号（正典＝表と数値の見せ方.md。ここは要約）

| 記号 | 意味 | 使う場所 |
|---|---|---|
| ●◐○ | 確度（高／中／低） | 課題仮説シートの仮説確度のみ |
| ◎○△ | テーラリング採否（採用／一部／省く） | 計画提案書・テーラリング記録のみ |
| 🟡 | 未確定 | 全成果物共通 |
| [要対応] | 意思決定者の判断が要る | 状況報告・終結報告書 |
| [引継ぎ] | 次フェーズ・運用へ渡す | 終結報告書 |
| [計画済]／[予期外] | 変更のトリガー区分 | 変更管理表 |

**禁止**: 同じ記号を別の軸に流用する（確度●を状態に、採否◎を達成度に等）。

## 4. 計画・実行の語

| 語 | 定義 |
|---|---|
| ベースライン | 計画で確定した約束（要求・スコープ・スケジュール・予算・計画方針）。変えるには変更管理表（#8）を通す。 |
| ローリングウェーブ | 直近を詳細化し、以降は条件付き骨格に留めてゲート後に詳細化する計画方式。 |
| 律速工程（クリティカルパス） | ここが遅れると全体が遅れる工程。計画で最重点に置く。 |
| ゲート | 段階の節目の判断点（続行／やり直し／中止）。「やめる自由」を組み込む。 |
| 受入基準 | 「何をもってできたとするか」の完了の線。 |
| テーラリング | PMBOKのプロセス・成果物を案件に合わせて採否する判断（採用/一部/省く＋理由）。全部はやらない。 |
| 不可逆リスク | 起きたら取り返しがつかないリスク（情報漏洩・統制違反等）。発生前提で先手を打つ。 |

## 5. 連動・台帳の語

| 語 | 定義 |
|---|---|
| リスク登録簿 | まだ起きていない不確実なこと（計画§3）。起きる前の手当てと兆候を持つ。消さない。 |
| 課題管理表（issue-log） | すでに起きている／いま手を打つべきこと。担当・期限を付けて閉じにいく。 |
| 変更管理表（change-log） | 計画のベースラインを変える要求だけ。影響評価と承認を通して計画へ反映する関所。 |
| ID連動 | R-xx（リスク）↔I-xx（課題）↔C-xx（変更）をリンクで指し、中身は一方にだけ書く（二重管理しない）。 |

## 6. 仕組み・運用の語

| 語 | 定義 |
|---|---|
| 子Skill／親Skill | 子＝成果物1つを作るSkill（知識を持つ）。親（オーケストレーター）＝順番・受け渡し・停止だけを持つ薄いSKILL.md。 |
| 伴走→写し取り | 机上設計せず、実案件で一度一緒に作ってから、その判断をSkillに写し取る作り方。 |
| 引き継ぎ書 | 前段→立ち上げの受け渡し1枚（凍結スナップショット）。固定パス `00_前段/引き継ぎ書_前段→立ち上げ.md`。 |
| project-context | 案件の「生きた本体」。確定事実の書き戻し先（案件ルート直下の1箇所）。 |
| ゴールデン／回帰eval | Skillの出力が崩れていないかを点検する固定の見本案件と、その点検（`_tools/eval`）。 |
| SSOT／ドリフト | 正典を1つに固定すること／コピーが正典とズレること。同期は `_tools/build.sh --sync`。 |
| Flow／Stock | 作業ドラフト＝`30_Flow`／完成・再利用ナレッジ＝`40_Stock`。 |
