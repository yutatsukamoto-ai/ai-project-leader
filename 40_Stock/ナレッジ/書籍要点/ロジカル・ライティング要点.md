## 0. 全体像：読み手にわかる3要件（第6章 / 図6-1）

| 要件 | 中身 | 崩れると |
|---|---|---|
| 1. 具体的に表現する | 物事の「中身」を表す／曖昧な言葉・表記を避ける | 何を言っているか伝わらない |
| 2. 論理的な関係を正しく表現する | MECEな関係を表す／So What?・Why So?の関係を表す | 筋が通らない・飛躍する |
| 3. 簡潔に表現する | 文の作りをシンプルに／無駄な表現を削る | 読むのに疲れる |

## 1. 書く前の設定（第1章 組み立ての準備）★最初にやる

紙に向かう前に「誰に・何の問いに・何をしてほしくて書くか」を決める。ここがズレると以降すべてズレる。

- **テーマを「問い」に置き換える** — 漠然としたテーマを、答えるべき具体的な問いに変換する。「何について書くか」でなく「どの問いに答えるか」。
- **期待する反応を決める** — 読み終えた相手に**何をしてほしいか**（理解／承認／行動）。ゴールの形が決まる。
- **読み手を確認する** — 目の前の相手だけでなく**「隠れた読み手」**（決裁者・後で読む人）はいないか。
- **書き手を確認する** — この文書の発信者は誰か（個人か組織か）。

> **空・雨・傘（コラム）**: アクションの説明はこの3点を含むべき。**空**＝事実（空が曇っている）→ **雨**＝解釈（ひと雨きそうだ）→ **傘**＝行動（傘を持って行こう）。事実だけ、解釈だけ、行動だけにならない。本プロジェクトの「提案＋理由」「認識確認してから動く」と同じ骨。

## 2. 論理の道具（第2章 本論の組み立て1・ロジカル・シンキング概説）

- **MECE** — 重複・漏れ・粒度のずれなく分ける。項目を並べたら必ず括って名前（見出し）をつける。
- **So What? / Why So?** — **So What?＝「だから何？」**（下の根拠を束ねると上の結論が言えるか）。**Why So?＝「なぜそう言える？」**（上の結論の下に根拠が足りているか）。この縦の問答が「筋が通る」の正体。
- **2つの論理パターン**:
  - **並列型** — 結論の下に、根拠を横に並べる（A・B・Cが対等）。
  - **解説型** — 事実 → 判断基準 → 判断、の順で1本道に説く。

## 3. 本論の組み立て・実践（第3章 / 図3-1・3-2）

- **2つの組み立て方**:
  - **結論から根拠へ** — 結論が固まり根拠も見えている場合。先に結論。
  - **根拠から結論へ** — 元情報から確定したい場合。手順は——
    1. **Step1** 問いを確認し、論理パターン（並列型／解説型）を選ぶ
    2. **Step2** 上から下へMECEに枠組みを作る
    3. **Step3** 下から上へ So What?／Why So? で縦の整合を検証する
- **セルフチェック3点（第3章3）**:
  - Check1 読み手の Why So? に過不足なく答えているか
  - Check2 要旨（結論）は明快か
  - Check3 結論を先に伝えるか、根拠を先に伝えるか（相手・状況で選ぶ）

## 4. 導入部の組み立て（第4章 / 図4-1〜4-4）

文書の入口。ここで読む気と理解の枠が決まる。

- **避けたい3タイプ（bad）**: ①埋没タイプ（結論が前置きに埋もれる）②表題の繰り返しタイプ（表題を言い直すだけ）③ご挨拶に終始タイプ。
- **good の作り**: 冒頭で「設定の共有（何の問いか）」＋「読み手の視点での棚卸し」を示し、結論へ最短で入る。図4-2・4-4は、結論先＋見出し分けで読み手が見出しだけで筋を追える。

## 5. 組み立ての視覚化（第5章 / 図5-2）★実務直結・最重要

頭の中の論理を、紙面の見た目へ翻訳する。「伝わる」の決め手。

- **Point1 表題・見出しを明記する** — 表題でテーマと期待する反応を示唆。見出しで組み立てを明示。**見出しは2型を使い分ける**: 〈項目型＝何の話かを示す〉と〈So What?型＝言いたい結論そのものを書く〉。
- **Point2 記号・スペースを活用する** — 箇条書き・インデント・余白で、論理上の位置（親子・並列）を目で見せる。**親の結論を太字で立て、内訳を字下げの入れ子に**。
- **Point3 文頭で説明の切り口を明示する** — 各項目の頭に「何の話か」のラベル（例「コスト削減：…」）。

構造の型:
```
★の検討結果のご報告        ← 表題（テーマ＋期待する反応）
  結論を表す見出し（導入部）  ← 結論
  ● AAA（根拠Aの見出し）     ← 本論
      - a-1 / a-2
  ● BBB（根拠Bの見出し）
      - b-1 / b-2
```

- **good実例（図5-8 申請書）**: 「3つの観点から参加を希望いたします」と結論を立て、〈トレンドの把握〉〈先進機関の対応例〉〈当部の課題への意見〉を太字ラベル＋説明の箇条書きで並べる。
- **bad実例（図5-7）**: 同じ欄を改行も見出しも無いズルズルの一段落で書く。
- **視覚化の応用（補論）**: 規定書式（申請書・テンプレの決まった枠）でも、枠内を結論＋箇条書きで視覚化する。プレゼン資料にも同じ3ポイントが効く（→将来のスライドSkillの基準）。

> 本プロジェクトの文章作法B「視覚化3ポイント」はこの章そのもの。チャーターv0.1 §10（「期間は全6ヶ月」を親、段階内訳を入れ子）が適用例。

## 6. 日本語表現の3要件の実装（第6章）

§0の3要件を文レベルで実装する。要件1 具体的に（中身を表す・曖昧/ジャーゴンを避ける）／要件2 論理的に（MECE・So What?/Why So?を文の関係で表す）／要件3 簡潔に（文をシンプルに・無駄を削る）。

## 7. セルフエディティング・チェックリスト（まとめ）

書き終えたら自分で点検する。**この点検は裏で行い、成果物には出さない**（文章作法D）。

- 設定: 答えるべき問いに答えているか／期待する反応に向いているか／隠れた読み手を外していないか
- 論理: So What?／Why So? が縦に通るか／MECEか／要旨は明快か
- 視覚化: 見出しだけで筋が追えるか／親子関係が見た目で分かるか
- 表現: 具体的か／簡潔か／ジャーゴンを無自覚に使っていないか

## 使い方

1. **書く前**: §1で設定（問い・反応・読み手）を固める。
2. **組むとき**: §2道具 → §3結論/根拠の組み立て → §5視覚化、の順。
3. **点検**: §7チェックリスト（裏で確認・出力には出さない）。
4. **学習（Why②）**: bad/good対比で「なぜ良いか」を言語化。
5. **将来の別Skill（スライド・レビュー）**: ここを共有ライブラリとして引く。
