# 書籍要点参照ルール

> 目的: 書籍の知識が「存在するだけで使われない」問題を防ぐ
> 作成日: 2026-06-19

---

## なぜこのルールが必要か

参考資料・書籍・ナレッジがいくら蓄積されても、成果物を作り始めるとき「汎用知識で書けそう」と判断して参照されないことがある。知識の「存在」と「参照タイミング」を接続しないと死蔵になる。

---

## ルール

### 1. 書籍の要点はMDにまとめる

書籍をナレッジ源泉として採用したら、必ず `40_Stock/ナレッジ/書籍要点/{書籍名}_要点.md` を作る。
フルで要約しなくてよい。**漏れてほしくない核心だけ**をまとめる。

### 2. 関連成果物・Skillに参照を埋め込む

書籍要点MDを作ったら、関連する成果物・Skillに参照トリガーを入れる。

| 場所 | 方法 |
|------|------|
| 成果物テンプレート（.md） | 冒頭に `> 参照: {要点MD パス}（作成前に必ず確認）` を入れる |
| SKILL.md の手順 | 入力を読む箇所に `{要点MD}` を追記する |
| SKILL.md の入口チェック | 関連書籍があれば「ヒアリングシート生成時は要点MDを参照する」を明示 |

### 3. 関連する話題が出たらClaudeから能動的に参照する

以下の場面では、対応する要点MDを確認してから作業する。

| 場面 | 参照先 |
|------|--------|
| PoC関連の成果物を作るとき | `書籍要点/実践PoC_要点.md` |
| ヒアリングシート（PoC型）を作るとき | 同上 |
| 停止ポイントで品質を問うとき（全フェーズ） | `書籍要点/プロジェクトリーダーの仕事_章まとめ要点.md` |
| 文章の論理構造に迷ったとき（MECE・So What?等） | `書籍要点/ロジカル・ライティング要点.md` |
| （他書籍追加時はここに追記） | |

## 書籍を追加するときの手順

1. 要点MDを `40_Stock/ナレッジ/書籍要点/{書籍名}_要点.md` に作成
2. この一覧表に追記
3. 関連する成果物テンプレート・SKILL.mdに参照トリガーを埋め込む
4. メモリーに「この書籍は○○のときに参照する」を記録する
