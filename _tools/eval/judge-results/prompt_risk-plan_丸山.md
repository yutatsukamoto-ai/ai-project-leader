## §1 リスクカテゴリ定義

本案件のリスクを4カテゴリに分類する。

| カテゴリ | 定義 | 本案件での代表例 |
|---------|------|----------------|
| 組織・人 | ステークホルダーの抵抗・不在・能力不足 | 村上氏の協力拒否、大西氏の判断遅延 |
| 定着・変化 | 改善施策の現場定着に関する不確実性 | IT不信による消極的抵抗、旧運用への回帰 |
| スコープ・外部 | スコープの混同、外部環境の変化 | 基幹入替との混同、●●工業の条件変更 |
| 技術・データ | 基幹データの質・取得可能性 | オービック7からのデータ抽出困難 |

## §2 評価基準

### 確率

| レベル | 定義 | 目安 |
|--------|------|------|
| 高 | 発生がほぼ見込まれる | 70%超。過去に類似事象あり |
| 中 | 条件次第で発生しうる | 30〜70%。兆候がある |
| 低 | 可能性は低いが無視できない | 30%未満。前提が崩れた場合 |

### 影響度

| レベル | 定義 |
|--------|------|
| 致命的 | プロジェクト目的（W3）の達成が不可能になる |
| 高 | 段階の遅延（1ヶ月超）または品質の大幅低下 |
| 中 | 段階内の調整で対処可能な遅延・追加コスト |
| 低 | 作業レベルの手戻り。全体への影響は軽微 |

### リスクスコア

確率×影響度で優先順位を決定。致命的は確率に関わらず最優先で管理。
