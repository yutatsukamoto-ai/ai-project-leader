## 目的

計画のベースライン（要求・スコープ・スケジュール・予算・計画方針）を変える要求だけを管理する関所。
思いつき反映を防ぎ、影響評価と承認を通してから計画へ反映する。

## 境界（載せる/載せない）

| 載せる | 載せない |
|--------|---------|
| 要求の追加・削除 | 指名・人選の判断待ち（→課題管理表） |
| スコープの変更（やること/やらないことの移動） | 作業課題（→課題管理表） |
| スケジュールBLの変更 | リスク対応アクション（→リスク登録簿） |
| 予算枠の変更 | 軽微な手順調整（BLに影響なし） |
| 計画方針の変更（進捗管理手法等） | — |

## 凡例

- **トリガー接頭辞**: [計画済] = 計画時に予見できた変更 / [予期外] = 想定外の要求
- **状態**: 起票 → 評価中 → 承認待ち → 承認 → 反映済 / 却下
- **関連**: 課題管理表（I-xx）・リスク登録簿（R-xx）・統合計画書の該当節へのリンク
