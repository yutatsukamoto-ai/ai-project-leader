## 共通チェック
- [状態は言葉] fail — Why表の確度欄が「●」「◐」「○」の記号のみで、言葉（高・中・低）が併記されていない。冒頭に凡例はあるが、チェック基準は「記号で代用していない」であり、記号のみの表記はこれに反する。
- [数字をぼかさない] pass — 「半年」「月末」等の期間表現はサマリーからの引用であり具体的。ただし新たな数値目標は設定されていないが、本成果物の性質（仮説段階）上妥当。
- [一文一改行] pass — 箇条書き・表形式・番号付きリストが中心で、一文一改行のルールが守られている。
- [ラベル型見出し] pass — 「Why — なぜこのプロジェクトをやるのか」「What — 何が課題か」「How — どう進めるか」「優先順位の仮説」と、見出しだけで構成を把握できる。
- [やらないこと・停止ポイントが明確] pass — 「あくまで仮説。現場ヒアリング・As-Is可視化で検証してから計画に落とす」と冒頭で位置づけを明言し、次工程（0-3リサーチメモ→0-4計画提案書）を明示している。
- [入力との整合] pass — 案件理解サマリーv0.2のW1〜W3、制約条件、過去失敗の情報と整合しており、矛盾は検出されない。
