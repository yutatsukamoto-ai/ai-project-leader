# 前段チェーン 最小実行パス

目的: 初回商談メモから、前段チェーンでどのように理解・仮説・提案・引き継ぎへ進むかを1本で追えるようにする。

素材の出どころ:

- `40_Stock/案件教訓/02_中堅IT企業_生成AI統制基盤.md` をもとにした脱識別済み模擬素材。
- 社名・個人名・金額・固有システム名は使わず、足りない前提だけ架空情報で補完。
- 実案件データではない。八束電工は本サンプルの作成対象ではなく、後続の実運用検証フィールドとして扱う。

読む順:

1. `00_入力_初回商談メモ.md`
2. `project-context.md`
3. `00_前段/01_案件理解サマリー_v0.2.md`
4. `00_前段/02_課題仮説シート_v0.1.md`
5. `00_前段/03_リサーチメモ_スキップ判断.md`
6. `00_前段/04_計画提案書_v0.1.md`
7. `00_前段/引き継ぎ書_前段→立ち上げ.md`

確認したいこと:

- 入力が薄い段階で、何を事実・推測・不明点に分けるか。
- 0-1の理解確認で止まる場所。
- 0-3リサーチを軽量化・スキップする判断理由。
- 0-4計画提案書の承認後、何を引き継ぎ書に運ぶか。
- `project-context.md` にどのタイミングで現在地・決定・次Skillを書き戻すか。

このサンプルの位置づけ:

- 正典サンプル: 前段チェーンの標準導線を示す。
- 品質検証用ではない: 実案件の完全な納品物ではなく、最小パスの見本。
- サンプル成果物インデックスでは、このフォルダを前段チェーンの代表サンプルとして扱う。
