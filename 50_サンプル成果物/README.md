# 50_サンプル成果物

AIプロジェクトリーダーの出力イメージを確認するための代表サンプル置き場。
ここは作業中ドラフトではなく、入口説明・品質目線合わせ・動作確認に使う最小代表だけを置く。

## まず見るもの

| 用途 | サンプル | 対応Skill / 領域 | 参照目的 |
|---|---|---|---|
| 前段チェーンの標準導線 | `md/前段_最小実行パス/` | `zendan-chain`、`anken-rikai-summary`、`kadai-kasetsu-sheet`、`keikaku-teiansho` | 初回商談メモから計画提案・引き継ぎまでの流れを見る |
| 成果物全体像 | `html/成果物マップ_showcase.html` / `.png` | 成果物マップ | フェーズ×成果物の全体像を見る |
| 課題管理表 | `xlsx/課題管理表_サンプル.xlsx` | `issue-log` | 実行フェーズの生きた台帳の粒度を見る |
| 週次状況報告 | `docx/規約テスト_週次状況報告サンプル.docx` | `status-report` | 客先共有文書の文体・構成を見る |
| AIプロジェクトリーダー紹介 | `pptx/20260617_AIプロジェクトリーダー紹介.pptx` | スライド系横断Skill | プロジェクト全体の説明スライド例を見る |

## Office / HTML サンプル

| フォルダ | 内容 | 使いどころ |
|---|---|---|
| `docx/` | 提案書・リサーチメモ・状況報告の文書サンプル | 文章作法、提出文書の粒度確認 |
| `xlsx/` | 課題管理表、月次売上サマリー | 生きた台帳や表形式の見え方確認 |
| `pptx/` | AI/DX提案・紹介・品質チェック用スライド | スライド系Skillの出力雰囲気確認 |
| `html/` | 成果物マップの閲覧版 | GitHub上で全体像を素早く見る |

## 追加ルール

- サンプルを大量に増やさない。代表として説明できるものだけを置く。
- 実案件データは置かない。必要な場合は脱識別し、出どころと補完範囲を明記する。
- 生成バイナリは最小代表に限る。作業中のレンダー、スクリーンショット、旧版ドラフトは `30_Flow/outputs/` 側で扱う。
- 次のSkill入力になる正典成果物の例は、`outputs/` ではなくこのフォルダかフェーズフォルダ側で管理する。
