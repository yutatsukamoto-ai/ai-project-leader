## バトンの流れ（一本道＋分岐）

案件理解(0-1) → 課題仮説(0-2) → リサーチ(0-3) → 計画提案(0-4)
　→ ［引き継ぎ書］ → チャーター(01) → 統合計画(02)
　→ ［テーラリングで必要な管理計画を選択: scope/communication/schedule/cost/resource/risk/PoC提案書］
　→ 課題管理＋コミュニケーション管理＋資源管理(03)
　→ 監視(04：状況報告＋監視カード5領域／必要時 変更管理) → 終結報告(05)

PoC型案件: 計画提案(0-4) → PoC提案書(02) → PoC実行(03) → PoC評価報告書(05) → Go判定 → 本番へ

監視は節目で回し、見つけた矛盾を課題管理（手を打つ）か変更管理（基準を変える）へ戻す。
