# コミュニケーション管理記録 作成・運用（子Skill）

実行フェーズの基幹成果物（#26 ステークホルダーエンゲージメントのマネジメント / #27 コミュニケーションのマネジメント）。
コミュニケーション計画（#24/#25）で設計した会議体・報告・エンゲージメント戦略を実行に移し、「実際に伝えて、効いているか」を追跡する生きた台帳を作る。
中身の品質基準・シート構成・連動ルールは `references/型と品質基準.md` を見る（Progressive Disclosure。本体は薄く保つ）。

## 位置づけ

コミュニケーション計画（計画フェーズ）が「何を・誰に・どう伝えるか」の設計図。
本Skillはその設計を実行に移し、**実績を記録し・効果を追跡し・ズレを検知して手を打つ**運用台帳にする。
計画は設計、本Skillは実行＋追跡。二重化せずリンクで持つ。

## 手順

1. **入力を読む**: `{案件名}/02_計画/NN_コミュニケーション計画_vX.md`・`{案件名}/01_立ち上げ/02_ステークホルダー登録簿_vX`・`project-context.md`。
   - コミュニケーション計画の✋承認が済か確認する。未なら止める。
   - 計画のコミュニケーションマトリクス（§2）と会議体設計（§3）が本Skillの**計画側の正典**。

2. **台帳のシートを立てる（停止・壁打ち）**: `references/型と品質基準.md` の構成に沿い、4シートを用意する。
   - **会議体運用ログ**: 計画§3の会議体を行に並べ、開催ごとに1行追加（日付・出席者・決定事項・アクション・議事録リンク）。
   - **エンゲージメント実績トラッカー**: 登録簿の重点管理対象を行に並べ、関与度（現在→目標）の実績を追跡する。
   - **情報配信実績**: 計画§2マトリクスの各行について、計画vs実績（配信日・手段・備考）を記録する。
   - **アクションアイテム管理**: 会議や対応から出たアクションを集約し、担当・期限・状態で追跡する。
   - 【差し込みカード参照枠】ドメインカードがあれば引く（無ければ素通り）。

3. **推測は根拠明示・未確定は🟡**: 関与度の変動評価や効果判定には根拠を添える。キックオフ前・人選未の行は🟡で残す。
   - 記号は `references/表と数値の見せ方.md` の正典どおり。状態は言葉（予定/実施済/未実施/取消）。確度●◐○・採否◎○△を流用しない。

4. **停止・壁打ち**: 計画vs実績のギャップが大きい行を巻き上げ、手を打つべき箇所を素材として出す。
   - 重点管理者の関与度が計画と乖離していたら、登録簿の更新とエンゲージメント戦略の見直しを促す。
   - 曖昧・不可逆な分岐が出たら `grill-me`（`20_Skills/90_横断/`）で詰める。

5. **検査ゲート（出力前・必須停止）**: 共通7項目＋本Skill固有を回す（共通ひな形§検査ゲート）。
   - 共通: ①入力✋確認済か ②必須項目・品質バー（未確定は🟡） ③記号は正典どおりか ④正典を二重化していないか ⑤出力先パス・版番号 ⑥未確認の明示（確認しなかった範囲＋理由を一覧） ⑦サボり回収ループ（スキップ理由を再検証、サボりはやり直し）。
   - 固有: ⑧計画のマトリクス全行に実績記録があるか ⑨重点管理者の関与度に変動がないか（変動あれば登録簿を更新） ⑩未消化アクションアイテムの棚卸しがされているか。
   - 結果を ✅／🟡／❌ で出し、最終判断は本人に返す。✋承認を得る。
   - **結果はレシート書式で必ず提示する**（チャットに出す＋`{案件名}/_検査ログ.md`へ1行追記）。

6. **承認後**: 確定事実（関与度の変動・会議体の変更等）を `project-context.md` へ書き戻す。
   - 出力先 `{案件名}/03_実行/NN_コミュニケーション管理_vX.xlsx`（生きた台帳＝md→xlsx）。壁打ち中はマークダウン、固まったらxlsx化。

## ルール

- **コミュニケーション計画との二重化をしない**。計画＝設計図、本台帳＝実績記録。計画の内容を転記して二重管理しない。計画の行を参照して実績を記録する（IDリンク）。
- **ステークホルダー登録簿との連動**: 関与度の変動を検知したら登録簿を更新する（本台帳に関与度のマスターを持たない。登録簿が正典）。
- **記号を流用しない**（`references/表と数値の見せ方.md`）。状態＝予定/実施済/未実施/取消。確度（●◐○）・採否（◎○△）はこの表で使わない。
- 文体は `references/文章作法.md`（6ルール）。一文一改行・ラベル見出し・🟡で未確定を残す。
- プロセス参照は正典 `10_参考資料/PMBOK第8版_40プロセス一覧.md` の #番号で行う（#26・#27）。
- 多段階で勝手に進めない。✋承認なしに出力（xlsx）を確定しない。各ステップ開始時に現在地を明示する（例:「実行 コミュニケーション管理記録」）。
