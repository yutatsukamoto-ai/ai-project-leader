# コミュニケーション管理記録 型と品質基準

出典: PMBOK第8版 #26 ステークホルダーエンゲージメントのマネジメント / #27 コミュニケーションのマネジメント。
位置づけ: SKILL.md本体（薄い）の品質バー。該当ステップでのみ読む（Progressive Disclosure）。

---

## この成果物の正体

コミュニケーション計画（#24/#25）で設計した仕組みを実行に移し、「伝えた・効いた・ズレた」を記録する生きた台帳。
計画が「何をどう伝えるか」の設計図なら、本台帳は**実行の実績記録＋効果追跡**。
品質バー＝「計画の全行に実績があるか」＋「関与度の変動を追跡しているか」＋「アクションが漏れなく管理されているか」＋「計画と二重化していないか」。

## 品質バー（このSkillの合格条件）

1. **計画のマトリクス全行に実績記録がある**（計画したのに追跡していない行がない）。
2. **重点管理者の関与度を追跡している**（登録簿の現在→目標に対する実績が見える）。
3. **未消化アクションアイテムの棚卸しがされている**（放置されたアクションがない）。
4. コミュニケーション計画と**二重化でなく実績記録・接続**になっている。
5. 情報が**届いているか・理解されているか**の効果確認がある（配信しただけで終わらない）。

## シート構成（4シート＋凡例）

### シート1: 会議体運用ログ

計画§3の会議体を管理し、開催ごとに1行追加する。

| 列 | 中身 | 注意 |
|---|---|---|
| 会議ID | MTG-01… | 会議体ごとの連番。 |
| 会議名 | キックオフ・進捗定例・ステコミ 等 | 計画§3と合わせる。 |
| 開催日 | YYYY-MM-DD | 予定日と実施日が異なれば両方。 |
| 出席者 | 名前（または役割） | 欠席者も記録（特に意思決定者の欠席）。 |
| 決定事項 | その会議で決まったこと | 事実で。一文ずつ。 |
| アクション | 会議から出た次の一手 | アクションアイテム管理（シート4）に起票。IDリンク。 |
| 議事録リンク | ファイルパスまたはURL | 議事録本体は別管理。ここはリンクのみ。 |
| 備考 | 予実差異・次回への申し送り | — |

### シート2: エンゲージメント実績トラッカー

登録簿の重点管理対象の関与度を定期的に追跡する。

| 列 | 中身 | 注意 |
|---|---|---|
| 対象者 | 登録簿からの重点管理者 | 登録簿のIDとリンク。 |
| 関与度 目標 | 登録簿の「目標」をそのまま引用 | 登録簿が正典。ここにはコピーしない＝参照。 |
| 関与度 実績（月次等） | 現時点の評価 | 〔不認識/抵抗/中立/支持/主導〕。評価日を併記。 |
| ギャップ | 目標との差 | ギャップがあれば対応策をアクションに起票。 |
| 対応実績 | 実際に打った手 | エンゲージメント計画の具体策との対照。 |
| 状態 | 追跡中/目標達成/要対応/保留 | 言葉。 |

**関与度に変動があったら、登録簿の「関与度 現在」を更新する**（本台帳は追跡記録、登録簿がマスター）。

### シート3: 情報配信実績

計画§2マトリクスの各行について計画vs実績を記録する。

| 列 | 中身 | 注意 |
|---|---|---|
| マトリクス行 | 計画§2の「対象」列をそのまま参照 | 計画側の行とIDリンク。 |
| 計画の頻度・手段 | 計画§2から参照 | 転記しない＝リンク。 |
| 直近の配信日 | YYYY-MM-DD | 予定日に対する実績。 |
| 配信状態 | 予定/実施済/未実施/取消 | 言葉。 |
| 効果確認 | 届いたか・理解されたか | フィードバック・反応の有無。「送っただけ」を防ぐ。 |
| 備考 | 計画との差異・改善点 | — |

### シート4: アクションアイテム管理

会議やエンゲージメント対応から出たアクションを集約する。

| 列 | 中身 | 注意 |
|---|---|---|
| ID | ACT-01… | 連番。消さない。 |
| アクション内容 | 何をするか | 一文で。具体的に。 |
| 発生元 | 会議ID（MTG-xx）・エンゲージメント対応 等 | トレーサビリティ。 |
| 担当 | 人名（仮置き時は役割） | 「みんな」は不可。一人に名指す。 |
| 期限 | YYYY-MM-DD | 未定なら相対表現＋確定タイミング。 |
| 状態 | オープン/対応中/完了 | 言葉。完了でも行は残す。 |
| 関連 | I-xx（課題管理表）等 | 課題管理表に連動するアクションはIDリンク。 |

### シート5: 凡例

状態の定義・シート間の連動ルール・計画との対照ルール・出典を載せる。

## コミュニケーション計画との連動ルール（最重要）

計画と管理記録は別物。計画＝設計図、管理記録＝実績台帳。混ぜず、リンクでつなぐ。

- **役割の違い**: 計画＝何を・誰に・どう伝えるかの設計（#24/#25）。管理記録＝実際に伝えて・効いているかの実績追跡（#26/#27）。
- **二重管理しない**: マトリクスの内容は計画にだけ持つ。管理記録には計画行へのリンクと実績を記録する。
- **変動の検知→登録簿更新**: エンゲージメント実績で関与度の変動を検知したら、**登録簿を更新する**（管理記録が関与度のマスターを持たない）。

## ステークホルダー登録簿との連動

- 登録簿＝関与度の正典（マスター）。
- 本台帳のエンゲージメント実績トラッカーは追跡記録。
- 変動を検知したら登録簿の「関与度 現在」を更新し、必要に応じてコミュニケーション計画の見直しを促す。

## 判断の分かれ目（人が確認すべき点。Skillはここで止まる/問う）

- **効果確認の深度**: 「送った」で終わるか「届いた・理解された・行動が変わった」まで追うかは案件の規模と重要度で決める。重点管理者には深く、様子見には最小限。
- **関与度変動の閾値**: どの程度の変動で「要対応」にするかは案件固有。抵抗→中立は改善、支持→中立は要注意。
- **会議体の追加・廃止**: 計画にない会議が必要になった場合、または計画の会議が不要になった場合は、計画の版を上げるか判断を返す。
- **可逆／不可逆の線引き**:
  - 可逆＝会議の頻度変更・配信手段の調整・アクションの優先度変更。やってみて直す。
  - 不可逆寄り＝対外的な報告体制の変更（顧客への約束変更）・重要な関係者への情報開示判断。✋を通す。

## やってはいけない

- コミュニケーション計画の内容を管理記録に転記して二重管理する（リンクで持つ）。
- 関与度のマスターを本台帳に持つ（登録簿が正典）。
- アクションを担当・期限なしで放置する（課題管理表と同じ原則）。
- 状態に確度記号（●◐○）・採否記号（◎○△）を流用する。
- 「配信した」だけで効果確認をしない（重点管理者には必ず効果確認）。
- 完了したアクションの行を消す（記録・振り返りの材料）。

## ✋承認前チェック

- [ ] 計画のマトリクス全行に実績記録があるか
- [ ] 重点管理者の関与度を追跡し、変動があれば登録簿を更新したか
- [ ] 未消化アクションアイテムの棚卸しがされているか
- [ ] コミュニケーション計画と二重化でなく実績記録・接続になっているか
- [ ] 会議体の開催記録（出席・決定事項・アクション）が漏れなくあるか
