# 資源管理記録 作成・運用（子Skill）

実行フェーズの基幹成果物（#32 資源の獲得 / #33 チームをリードする）。
資源管理計画（#30/#31）で設計した体制・責任分担・チームチャーターを実行に移し、「チームが動いているか・資源が足りているか」を追跡する生きた台帳を作る。
中身の品質基準・シート構成・連動ルールは `references/型と品質基準.md` を見る（Progressive Disclosure。本体は薄く保つ）。

## 位置づけ

資源管理計画（計画フェーズ）が「誰を・どう配置するか」の設計図。
本Skillはその設計を実行に移し、**実際にチームを動かし・人を育て・問題を解決する**運用台帳にする。
計画は設計、本Skillは実行＋追跡。二重化せずリンクで持つ。
※正典の表記は「資源」（リソースではない）。

## 手順

1. **入力を読む**: `{案件名}/02_計画/NN_資源管理計画_vX.md`・`project-context.md`。
   - 資源管理計画の✋承認が済か確認する。未なら止める。
   - 計画の体制図（§2）・RACIマトリクス（§3）・チームチャーター（§4）・資源カレンダー（§5）が本Skillの**計画側の正典**。

2. **台帳のシートを立てる（停止・壁打ち）**: `references/型と品質基準.md` の構成に沿い、4シートを用意する。
   - **資源獲得状況**: 計画§1の資源見積もりを行に並べ、計画vs実績の対照を記録する。
   - **チーム運営ログ**: 1on1・チームビルディング・研修・メンタリングの実績を記録する。
   - **稼働実績トラッカー**: 計画§5資源カレンダーの稼働率vs実績を追跡する。
   - **コンフリクト・パフォーマンス記録**: チーム運営上の課題と対応を記録する。
   - 【差し込みカード参照枠】ドメインカードがあれば引く（無ければ素通り）。

3. **推測は根拠明示・未確定は🟡**: 稼働率の評価やパフォーマンス判定には根拠を添える。アサイン未確定・外注先未選定の行は🟡で残す。
   - 記号は `references/表と数値の見せ方.md` の正典どおり。状態は言葉（未着手/進行中/完了/遅延/中止）。確度●◐○・採否◎○△を流用しない。

4. **停止・壁打ち**: 計画vs実績のギャップが大きい行を巻き上げ、手を打つべき箇所を素材として出す。
   - 稼働実績が計画と大きく乖離していたら、資源カレンダーの見直しとリスク登録簿への起票を促す。
   - コンフリクト・パフォーマンス課題で不可逆な判断（人の異動・契約解除等）が出たら✋で止める。
   - 曖昧・不可逆な分岐が出たら `grill-me`（`20_Skills/90_横断/`）で詰める。

5. **検査ゲート（出力前・必須停止）**: 共通7項目＋本Skill固有を回す（共通ひな形§検査ゲート）。
   - 共通: ①入力✋確認済か ②必須項目・品質バー（未確定は🟡） ③記号は正典どおりか ④正典を二重化していないか ⑤出力先パス・版番号 ⑥未確認の明示（確認しなかった範囲＋理由を一覧） ⑦サボり回収ループ（スキップ理由を再検証、サボりはやり直し）。
   - 固有: ⑧全ロールのアサイン状況が最新か ⑨稼働実績が計画と乖離していないか ⑩コンフリクト・パフォーマンス課題に対応記録があるか。
   - 結果を ✅／🟡／❌ で出し、最終判断は本人に返す。✋承認を得る。
   - **結果はレシート書式で必ず提示する**（チャットに出す＋`{案件名}/_検査ログ.md`へ1行追記）。

6. **承認後**: 確定事実（アサイン確定・稼働率の変更等）を `project-context.md` へ書き戻す。
   - 出力先 `{案件名}/03_実行/NN_資源管理_vX.xlsx`（生きた台帳＝md→xlsx）。壁打ち中はマークダウン、固まったらxlsx化。

## ルール

- **資源管理計画との二重化をしない**。計画＝設計図、本台帳＝実績記録。計画の内容を転記して二重管理しない。計画の行を参照して実績を記録する（IDリンク）。
- **RACI実績は計画との照合で持つ**: RACIマトリクス本体は計画§3が正典。本台帳では「計画どおりに回っているか」の実績・例外を記録する。
- **記号を流用しない**（`references/表と数値の見せ方.md`）。状態＝未着手/進行中/完了/遅延/中止。確度（●◐○）・採否（◎○△）はこの表で使わない。
- 文体は `references/文章作法.md`（6ルール）。一文一改行・ラベル見出し・🟡で未確定を残す。
- プロセス参照は `references/PMBOK第8版_40プロセス一覧.md` の #番号で行う（#32・#33）。
- 多段階で勝手に進めない。✋承認なしに出力（xlsx）を確定しない。各ステップ開始時に現在地を明示する（例:「実行 資源管理記録」）。

## 共通ひな形の枠

正典は `40_Stock/横断ガイドライン/成果物Skill共通ひな形.md`。

- **差し込みカード参照枠**: ドメインカードがあれば手順「中身を埋める」で引く（無ければ素通り）。
- **検査ゲート（出力前・✋承認の直前）**: 共通7項目＋固有3項目を回す → 結果を ✅／🟡／❌ で出し、最終判断は本人に返す。 **結果はレシート書式で必ず提示する**（共通ひな形§レシート＝チャットに出す＋`{案件名}/_検査ログ.md`へ1行追記・客先提出物の本体には埋めない）。
