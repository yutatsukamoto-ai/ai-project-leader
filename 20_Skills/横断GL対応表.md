# 横断GL対応表

目的: Skill実行時に、どの横断ガイドラインを読むべきかを迷わないためのルーティング表。
`成果物マップ.md` が「どのSkillで何を作るか」の地図で、本表は「そのSkillでどの共通知識を読むか」の地図。

> 位置づけ: 横断ガイドライン本文の正典ではない。正典は `40_Stock/横断ガイドライン/` 配下。
> 本表は、全読みを避けつつ必要なルールへ到達するための索引。

---

## 使い方

1. `20_Skills/成果物マップ.md` で対象Skillを特定する。
2. 本表で、対象Skillの `常時読む` と `条件付きで読む` を確認する。
3. 対象Skillの `SKILL.md` と `references/` を読む。
4. 横断GLは、本表で必要になったものだけ読む。

chainで複数Skillを連続実行する場合、同じ横断GLをすでに読んでいて内容が変わっていないなら、再読せず「既読前提」でよい。
ただし、出力形式や用途が変わるとき（例: Markdown正典から客先提出docxへ変換、スライド化、Excel化）は、対応する条件付きGLを読む。

---

## 読み込みプロファイル

Skill実行時は、参照ファイルを次の4層に分ける。

| 層 | 読むもの | 読むタイミング |
|---|---|---|
| A. 入力・Skill本体 | 対象 `SKILL.md`、入力成果物、`project-context.md`、対象Skillの `references/型と品質基準.md` | 常時。ここが成果物の主たる型と案件文脈 |
| B. 横断必読 | 本表の `常時読む` にある横断GL、または同内容のSkill内 `references/` コピー | 常時。すでに同一chain内で読んだ同内容は既読前提でよい |
| C. 条件付き | 本表の `条件付きで読む` にある横断GL、Skill固有referencesのうち分岐時だけ使うもの | 出力形式・用途・判断分岐が該当した時だけ読む |
| D. 配布同梱・通常不要 | `.skill` 自己完結のため同梱されているが、今回の成果物には使わないreferences | 原則読まない。QA・変換・配布・構造変更で必要になった時だけ読む |

このプロファイルの狙いは、`.skill` に入っているファイルをすべて読むことではなく、**必要な知識へ到達しつつ、不要な再読を避ける**こと。
新規Skillを作る場合は、`SKILL.md` の冒頭または `## 参照ファイル` に A〜D の分け方を短く明記する。

---

## chain実行時の既読運用

chainで複数Skillを連続実行する場合、親chainは今回の会話・作業中だけ使う `既読横断GL` を持ってよい。
これは成果物でも監査ログでもなく、同じ横断GLを何度も読まないための作業メモ。

運用ルール:

1. chain開始時に、本表の親chain行を読む。
2. 子Skillを起動するたびに、本表の子Skill行で `常時読む` と `条件付きで読む` を確認する。
3. 同一chain内ですでに読んだ同一横断GLは、内容が変わっていなければ再読せず「既読前提」でよい。
4. ただし、出力形式・用途・提出先が変わる場合（例: docx/PDF化、Excel化、スライド化、客先共有）や、横断GLそのものを編集した後は、該当GLを読み直す。
5. `既読横断GL` は `chain-trace.json` に書かない。traceはSkill実行・承認・フェーズ遷移の索引に留める。

## 凡例

| 区分 | 意味 |
|---|---|
| 常時読む | そのSkillを実行するたびに必要。Skill内コピーがある場合は `references/` 側で読んでよい |
| 条件付きで読む | 出力形式・用途・判断分岐に該当するときだけ読む |
| 通常不要 | 原則読まない。構造変更・配布・特殊出力時だけ見る |

## chain / オーケストレーター

chainは成果物本文を作らず、順番・受け渡し・停止ポイントを管理する。
原則として成果物用の文章作法や表と数値は読まない。

## 更新ルール

この表を更新する条件:

- Skillを追加・改名・削除したとき。
- Skillの出力形式を変えたとき。
- 横断ガイドラインを新設したとき。
- 既存Skillが新しい横断GLを常時参照するようになったとき。
- 「読まなくてもよい」と判断していたGLが、実運用で必要になったとき。

更新時は、必要に応じて以下も確認する。

- `20_Skills/成果物マップ.md`
- `40_Stock/横断ガイドライン/README.md`
- `AGENTS.md`
- `CODEX.md`
- `README.md`
