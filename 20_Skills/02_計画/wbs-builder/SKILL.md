# WBS 作成（子Skill）

計画フェーズの成果物（#13 スコープ構造の構築）。
統合計画書§2スコープ（簡易WBS）を深掘りし、**事実（予定/実績の日付）だけを持つ `wbs.json`** にする。工数・進捗・遅延（イナズマ線）は描画側で自動計算＝手でメンテする数字を持たない。
中身の品質基準・データ形式・判断の分かれ目は `references/型と品質基準.md` を見る（Progressive Disclosure。本体は薄く保つ）。

## 出力と運用の方針（master=wbs.json／日常=Excel・節目=HTML）

- **master＝`wbs.json`**（事実だけ・AI保守）。Excelもイナズマ線HTMLも、この単一データから出し分ける。
- **日常の共有・閲覧＝Excel/Sheets**（誰でも低摩擦で開ける。Microsoft/Google環境に強い）。
- **節目（週次状況報告・レビュー）＝HTML(イナズマ線)** を生成し遅延を一目で見せる。確定配るPDFは不要。
- 更新は「作業者は報告→1人(PM/AI)が`wbs.json`を更新→各ビューを再生成」。報告は `40_Stock/テンプレート/WBS_更新依頼テンプレ.md`。
- 詳細・弱点対応・**状況報告(status-report)への差し込み**は `references/運用ガイド.md`。

## 手順

1. **入力を読む**: `{案件名}/02_計画/01_統合計画書_vX.md`（§2スコープ＝段階・主な作業→成果物・受入基準・律速）・`project-context.md`。
   - 統合計画書の✋承認が済か確認する。未なら止める。
   - **統合計画書§2は残す**。WBSはそれを深掘りした別物＝二重化しない（リンクで繋ぐ）。

2. **wbs.jsonを組む**: `references/型と品質基準.md` の形式・品質バーに沿う。
   - 段階→タスクを**最大3階層**（`children`あり＝集計／なし＝リーフ＝工数を持つ）。
   - **ローリングウェーブ**＝直近段階のみリーフまで詳細＋仮の絶対日付、先の段階は集計骨格＋日付null。
   - 律速（クリティカルパス）は該当リーフの `note` に「★律速」。不可逆の関所もnoteに明記。
   - 【差し込みカード参照枠】`40_Stock/カード/レシピ_作成/WBSの作り方.md` があれば引く。

3. **推測の根拠・🟡**: 未確定の日付・担当は `null`、確定タイミング（どのゲートで決めるか）をnoteか申し送りに。
   - 工数は素データ（`qty`×`hours`）のみ持つ。工数(人日)＝`qty×hours÷8`は自動計算＝持たせない。
   - 記号は `references/表と数値の見せ方.md` の正典（状態は言葉。●◐○/◎○△を流用しない）。

4. **停止・壁打ち（判断点）**: 粒度（どこまで割るか）／**日付の置き方**（直近段階のみ仮日付・先null が既定）／担当・工数の概算／律速の置き場。ここで止めて本人に振る。

5. **検査ゲート（出力前・必須停止）**: 共通5項目＋本Skill固有を回す（共通ひな形§検査ゲート）。
   - 共通: ①統合計画書✋確認済か ②必須項目・品質バー（未確定はnull/🟡） ③記号は正典どおりか ④§2と二重化でなく深掘りか ⑤出力先パス・版番号。
   - 固有: 3階層以内か／工数は素データのみで自動計算に委ねているか／日付は`YYYY-MM-DD`か／段階と受入基準が整合しているか／律速が示されているか。
   - 結果を ✅／🟡／❌ で出し、最終判断は本人に返す。✋承認を得る。

6. **承認後（成果物セットを出す）**:
   - `wbs.json` を確定（出力先 `{案件名}/02_計画/02_WBS_vX.json`）。
   - **閲覧版HTML**を生成（`assets/閲覧版テンプレ.html` の `DATA` をこの案件のwbs.jsonで置換 → `02_WBS_vX_閲覧版.html`）。ダブルクリックで見られる共有用。
   - **WBS Viewer（MIT）** を案内（編集・イナズマ線運用用）。入手・配置・帰属・閲覧版との使い分けは `references/WBS Viewerの入手と配置.md`。
   - **更新依頼テンプレ**（`40_Stock/テンプレート/WBS_更新依頼テンプレ.md`）の場所を案内。
   - 確定事実を `project-context.md` へ書き戻す。

## ルール

- データ形式は `wbs.json`（`references/型と品質基準.md`。上流仕様は WBS Viewer の `CLAUDE.md`）。**事実だけを持つ**。
- 文体は `references/文章作法.md`、記号は `references/表と数値の見せ方.md` の正典。
- プロセス参照は正典 `10_参考資料/PMBOK第8版_40プロセス一覧.md` の #番号（#13・日程の素地は#16-17簡易）。
- 統合計画書§2との重複は許容するが**二重化はしない**（§2＝簡易の土台、WBS＝深掘り台帳。リンクで繋ぐ）。
- **WBS Viewer は外部MIT製品**。同梱・再配布時は帰属を明記する（不可逆ガード）。
- 多段階で勝手に進めない。✋承認なしに次工程へ渡さない。各ステップで現在地を明示（例:「計画 02／WBS」）。
