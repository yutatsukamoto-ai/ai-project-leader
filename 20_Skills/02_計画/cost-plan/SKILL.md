# コスト管理計画 作成（子Skill）

計画フェーズの基幹成果物（#19 財務マネジメントの計画 / #20 コストの見積り / #21 予算の作成）。
統合計画書の概算コストとWBSのワークパッケージを入力に、**コスト見積もり＋コストベースライン＋管理ルール**を一体化した計画にする。
中身の品質基準・節構成は `references/型と品質基準.md` を見る（Progressive Disclosure。本体は薄く保つ）。

## 位置づけ

統合計画書が概算レベルの予算を持つ場合の**精緻化**。
予算が承認・説明の論点になる案件（外部ステークホルダーへの予算承認が必要、複数ベンダーの見積もり比較がある等）で起動する。
WBS（wbs.json）のワークパッケージをコスト積み上げの最小単位として使う。

## 手順

1. **入力を読む**: `{案件名}/02_計画/01_統合計画書_vX.md`・`{案件名}/02_計画/NN_wbs.json`・`project-context.md`。
   - 統合計画書の✋承認が済か確認する。未なら止める。
   - WBSが無い場合は統合計画書のスコープ記述から概算で進めてよいが、精度が落ちる旨を🟡で明示する。
   - `project-context.md` が無ければ止めず、承認後の書き戻し（手順6）で新規作成する。

2. **コスト見積もりを組む**: `references/型と品質基準.md` の型に沿う。
   - WBSのワークパッケージごとに見積技法（類推・パラメトリック・ボトムアップ等）を選定し、根拠とともに記載する。
   - 【差し込みカード参照枠】ドメインカードがあれば引く（無ければ素通り）。

3. **コストベースラインを設定する**: ワークパッケージ別見積もりを時系列で積み上げ、S字カーブの基礎データとする。
   - コンティンジェンシーリザーブ（リスク対応用・PM権限内）とマネジメントリザーブ（未知リスク用・スポンサー権限）を区別して設定する。
   - リザーブの根拠（リスク登録簿との対応、過去実績等）を明記する。

4. **推測には根拠を明示**: 見積もりの精度レベル（概算±50%〜確定±10%等）を添える。未確定（ベンダー見積もり未取得等）は🟡で残す。
   - 記号は `references/表と数値の見せ方.md` の正典どおり。

5. **停止・壁打ち**: EVM閾値（CPI/SPIの警告・是正ライン）の素案を出す。
   - 曖昧・不可逆な分岐が出たら `grill-me`（`20_Skills/90_横断/`）で詰める。

6. **検査ゲート（出力前・必須停止）**: 共通7項目＋本Skill固有を回す（共通ひな形§検査ゲート）。
   - 共通: ①入力✋確認済か ②必須項目・品質バー（未確定は🟡） ③記号は正典どおりか ④正典を二重化していないか ⑤出力先パス・版番号 ⑥未確認の明示（確認しなかった範囲＋理由を一覧） ⑦サボり回収ループ（スキップ理由を再検証、サボりはやり直し）。
   - 固有: ⑧WBSの全ワークパッケージにコスト見積もりがあるか ⑨コンティンジェンシーリザーブの根拠があるか ⑩EVM閾値が定義されているか。
   - 結果を ✅／🟡／❌ で出し、最終判断は本人に返す。✋承認を得る。
   - **結果はレシート書式で必ず提示する**（チャットに出す＋`{案件名}/_検査ログ.md`へ1行追記）。

7. **承認後**: 確定事実を `project-context.md` へ書き戻す。
   - 出力先 `{案件名}/02_計画/NN_コスト管理計画_vX.md`。

## ルール

- 文体は `references/文章作法.md`（6ルール）に従う。
- 記号は `references/表と数値の見せ方.md` の正典（他軸へ流用しない）。
- プロセス参照は `references/PMBOK第8版_40プロセス一覧.md` の #番号で行う（#19/#20/#21）。
- 統合計画書・WBSとの**二重化をしない**＝統合計画書は概算と前提、WBSはスコープ構造、本計画はコスト見積もり・ベースライン・管理ルール。役割を分けリンクで繋ぐ。
- 多段階で勝手に進めない。✋承認なしに次工程／提出版を確定しない。
- 各ステップ開始時に現在地を明示する（例:「計画 コスト管理計画」）。

## chain-trace.json

- 案件ルートに `chain-trace.json` がある場合は、検査ゲート通過後・✋承認後に本Skillのエントリを追記する（規約: `40_Stock/横断ガイドライン/チェーントレース規約.md`）。無い場合（前段のみ等）はスキップしてよい。
