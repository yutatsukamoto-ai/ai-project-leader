# 状況報告 作成（子Skill）

監視・コントロールフェーズの基幹成果物（#7 パフォーマンスの監視・コントロール）。
実行で出た事実（課題・リスク・進捗）を巻き上げて統合し、意思決定者が次の一手を判断できる一枚にする。
中身の品質基準・節構成・巻き上げの引きどころは `references/型と品質基準.md` を見る（Progressive Disclosure。本体は薄く保つ）。

## 手順

1. **入力を読む**: `{案件名}/03_実行/01_課題管理表_vX`・`02_計画/01_統合計画書_vX.md`（§3リスク登録簿）・`project-context.md`。
   - 報告対象の期間（週次なら今週）と、いま居る段階（ローリングウェーブのどこか）を確認する。

2. **巻き上げる（停止・壁打ち）**: 下流から統合の素材を引く。`references/型と品質基準.md` の引きどころに沿う。
   - 課題管理表の**高優先度・オープン** → 要対応事項。律速の課題を先頭に。
   - リスク登録簿の**状態（対応へ移管／顕在化済／オープン）** → リスクの動き。課題側IDと連動表示。
   - **ゲートの通過条件** → 次の意思決定。
   - **WBS（あれば・`02_計画/02_WBS_vX.json`）** → 律速の進捗・遅延（イナズマ線で左へ折れている行）・直近マイルストーンを要約。必要ならイナズマのスクショを添付（見る人はDL不要で状態が分かる＝wbs-builderの運用ガイドBの差し込み）。
   - 領域ごとに状態を一言で要約。**【差し込みカード参照】**各領域の監視カード（`40_Stock/カード/症状_源泉/`）があれば引いて、チェック指標で順調/注意/要対応を判定する:
     - スコープ → `スコープ監視の観点.md`（#14/#15）
     - スケジュール → `スケジュール監視の観点.md`（#18）
     - コスト → `コスト監視の観点.md`（#22）※予算が論点の案件のみ
     - ステークホルダー → `ステークホルダー監視の観点.md`（#28/#29）
     - リソース → `リソース監視の観点.md`（#34）
   - カードの症状に該当する兆候があれば、処方を§要対応事項に含める。カードが無い領域は素通り。

3. **一枚に組む**: 物語の順（全体像→詳細→次）で6節に収める。
   - サマリー（2文＋ダッシュボード1行）／領域別の状態／進捗ハイライト／要対応事項／リスクの動き（**動きがある週のみ**）／来週の予定と次の意思決定。
   - 週次は重くしない。1枚〜2枚で読み切れる粒度。テーラリングで要らない節は間引く。

4. **出力（停止・承認後）**: 最終形は **Word（.docx）**＝ある時点の提出・コミュニケーション文書。
   - 体裁: A4・游ゴシック・見出しに色＋下線・表ヘッダーに薄い背景・状態セルを色分け。
   - 運用: マスターはマークダウン、提出用docx変換は人の確認後（アジェンダ・議事録規格の運用原則に揃える）。
   - 出力先 `{案件名}/04_監視コントロール/01_週次状況報告_vX.docx`。

## ルール

- **生きた台帳でなく、ある時点のコミュニケーション文書**。課題管理表（Excel・台帳）と役割が違う。状況報告は読ませて意思決定につなげる物語型。
- **状態は言葉**（順調／注意／要対応）。確度（●◐○）・採否（◎○△）を状態に流用しない（`references/表と数値の見せ方.md`）。※過去サンプルが状態列に◎○△を使っていたのは誤り。言葉に直す。
- **数字をぼかさない**。課題件数・期日・ゲート日は具体で書く。意思決定が要る項目は [要対応] を頭に付ける。
- 文体は `references/文章作法.md`（6ルール）。一文一改行・ラベル見出し。Word内では絵文字を文字に置換（[要対応] 等）。
- プロセス参照は `references/PMBOK第8版_40プロセス一覧.md` の #番号で行う（#7）。
- 多段階で勝手に進めない。✋承認なしに提出版（docx）を確定しない。各ステップ開始時に現在地を明示する（例:「監視・コントロール 01／状況報告」）。

## 確定した運用ルール（2026-06-14）

- **§「リスクの動き」は動きがある週だけ載せる**。新規／対応へ移管／顕在化／クローズのいずれかがあった週のみ。静かな週は§「領域別の状態」のリスク行で足りる。これで§2との重複感を解消する。
- **ダッシュボード1行は3指標で固定**: スケジュール状態（順調/注意/要対応）／高優先度オープン件数／次の意思決定（ゲート＋期日）。増やさない。成果物充足の件数を載せる場合は「（本報告書除く）」を注記する。
- **ヘッダーの確定要素**: 案件名／報告日（YYYY-MM-DD）／第N週・段階／報告者。この並びを標準にする。
