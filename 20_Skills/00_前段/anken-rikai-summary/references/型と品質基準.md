# 案件理解サマリー 型と品質基準

出典: 模擬案件・山藤商事の試運転（2026-06-11〜12）で実証した型。修正ログ・05振り返りより写し取り。

## 文書の型（セクション構成）

1. **クライアント概要** — 公開情報ベース（業種・規模・トップの状況）
2. **相談の経緯** — なぜ相談に至ったか
3. **期待の構造化** — 発言から期待を表に分解（W1, W2...と採番）。各期待に「性質」を付ける（業務効率化／経営の見える化／手段先行 など）
4. **制約・条件** — 期間・予算など。**必ず「出どころ」を記録する**（下記参照）
5. **事実と推測の区別** — 発言で裏付けあり＝事実、要検証＝推測を明示
6. **不明点 → ヒアリング質問** — 優先度順
7. **ヒアリング結果**（v0.2で追加）— 信頼度記号付き+「ヒアリングから得た示唆」+「次回ヒアリング先候補」

## 判断基準（作成時に必ず行う思考）

- **期待の分解**: 「AIですごいこと」のような曖昧な期待は、必ず複数の期待が混在していると疑う。性質の異なる期待に分解する
- **手段と目的の区別**: クライアントが口にする手段（AI・ツール名）を目的扱いしない
- **最重要発言の特定**: クライアントが唯一明確に語った不安・条件（例:「現場がついてこないやつは困る」）を見つけ、以降の全成果物の軸にする
- **制約の出どころ3パターン**: ①クライアント指定（従う+妥当性検証）②未定（こちらが設計・提示）③自社側制約（スコープを削って収める）。どれかを必ず記録——0-4の建付けがこれで変わる
- **初期対象と対象外の分離**: 初期スコープ、将来要望、対象外、禁止・制約を混同しない。高リスク領域（誤回答、権限超え、外部公開、個人情報など）は「初期対象外」または「要確認」に逃がす
- **未確認情報の扱い**: 入力記載あり、要確認、未入力を分ける。聞き取り確度が弱い情報、添付未読、相対日付、件数・管理者不明などを事実として滑らかに断定しない
- **成功基準の拾い上げ**: 「何が見えたら成果か」を、発言から拾うか質問に回す。数値が無い場合は、探索時間、自己解決率、問い合わせ件数など測れる候補を仮置きする
- **資料・発言の鮮度確認**: 作成日、版、最終更新、現在も有効か、誰が確認したかを確認対象にする。古い資料や途中発言を最新合意として扱わない
- **定量主張の根拠確認**: 削減率、削減時間、費用対効果などの数字は、現状実測、類似案件実績、試算式のどれに基づくかを質問に回す
- **質問リストの優先度**: 過去の失敗・最大リスクに関わる質問を最優先。次に業務実態、成功イメージ

## 入力形式ごとの注意

- **口頭・録音文字起こし**: フィラー、言い直し、雑談を成果物本文に混ぜない。話者不明・「たしか」「気がする」などの弱い情報は要確認に回す。相対日付（来月、秋、年度内）は絶対日付の確認質問にする
- **メールスレッド**: 古い発言と後続メールの合意を分ける。主依頼者、情報システム、法務など発言者の役割で制約の重みを判断する。添付が入力に無い場合は未確認範囲として明示する
- **スライド・PDF**: 資料に書かれた事実と、資料から推測した意図を分ける。資料外の背景、意思決定者、最新性、作成目的は質問に回す

## 反復構造（このSkillの本質）

- v0.1は「事実/推測の区別+質問リスト付き」で出す。完成版を装わない
- ✋理解確認+ヒアリング → 回答を反映してv0.2
- 回答の信頼度記号: 【談】=本人談（要裏取り） 【未】=本人も不明
- **「回答者が知らないこと」自体が重要な発見**（例: 社長が現場実態を未把握 → 現状可視化フェーズの根拠になる）

## 合格チェック

- [ ] 期待が複数に分解され、性質が付いているか
- [ ] 手段と目的が区別されているか
- [ ] 制約に出どころ（①②③）が付いているか
- [ ] 事実と推測が区別されているか
- [ ] 初期対象、将来要望、対象外、禁止・制約が混同されていないか
- [ ] 誤回答、情報公開範囲、権限、運用不全など高リスク条件を拾っているか
- [ ] 成功基準を拾っているか。未確定なら確認質問に回しているか
- [ ] 添付欠落、日付不明、件数不明、管理者不明などの不足入力を質問に回しているか
- [ ] 作成日、版、最終更新、現在も有効か、誰が確認したかを質問に回しているか
- [ ] 削減率、削減時間、費用対効果など定量主張の根拠を確認対象にしているか
- [ ] 弱い情報（聞き取り不確実、添付未読、合意境界が曖昧な内容）を事実扱いしていないか
- [ ] 質問リストが優先度順で、次回ヒアリングに使える粒度か
- [ ] （v0.2）知らないこと・分からないことが発見として記録されているか
