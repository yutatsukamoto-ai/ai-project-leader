# project-context.md（生きた本体）

全Skillがこのファイルを読む。前提を毎回口頭で伝え直さないための一元管理。

更新ルール:

- 各工程で新事実・承認・方針が確定したら追記する。
- 推測は書かない。仮説は各成果物の中で管理し、確定したものだけここへ移す。
- 引き継ぎ書は凍結スナップショット。新事実は引き継ぎ書ではなく、このファイルへ書き戻す。
- 既存案件へ無理に遡及しない。新規案件・後続フェーズ着手案件からこの型を使う。
