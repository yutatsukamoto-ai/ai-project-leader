# リサーチメモ（前段チェーン 0-3）

## 目的

課題仮説の確度を上げ、計画提案書の説得材料（同業の事例・数字）を集めること。
このSkillの価値は「結論を変える」ことではなく**「確度昇格+説得材料の獲得」**にある。

## 入力

- 確認済みの `02_課題仮説シート`（最新版）+ `01_案件理解サマリー`（業種・制約の参照用）

## 手順

1. `references/観点とスキップ基準.md` を読み、**スキップ判断をユーザーに提示**（やる/軽く/スキップ+理由）
2. 実施する場合: **3層の検索をサブエージェント（Agentツール）で並列実行**（詳細は下記§サブエージェント設計）
3. サブエージェントの中間ファイル3本を統合し、メモを作成（文体は `references/文章作法.md` 準拠。出典リンクを必ず明記）
4. 提示+軽い確認: 「直したい点はありますか？」
5. OK後、次工程（0-4 計画提案書）へ受け渡し

## サブエージェント設計（手順2の詳細）

設計基準: `40_Stock/横断ガイドライン/サブエージェント設計基準.md` に準拠。

### なぜ並列化するか

3層の検索軸（同業種の典型課題 / 打ち手事例 / 失敗事例）は互いに独立しており、直列にする理由がない。
並列化により速度が上がり、メインのコンテキストが検索結果で圧迫されない（原則A-2, A-3）。

### 起動方法

手順2で、Agentツールを**1つのメッセージ内に3つ同時に**呼び出す（並列実行）。
各エージェントは1つの検索層のみ担当する（原則A-1）。

### 各エージェントの仕様

**共通ルール（3エージェント共通でpromptに含める）:**
- 案件の業種・課題仮説のコンテキストを渡す（案件理解サマリー+課題仮説シートの要点を転記）
- 結果はファイルに書き出す（原則A-3）。パスは `outputs/research_layer{1,2,3}.md`
- メインへの戻り値は「○件の事例を収集。詳細はファイル参照」程度の要約のみ
- 「5回以内のWeb検索で完了せよ」と明記（原則A-4: 暴走防止）
- 各事実に出典リンクを必ず付ける
- ベンダー記事のバイアスに注意。数字は「事例として紹介」レベル

| エージェント | description | 検索層 | 出力ファイル |
|---|---|---|---|
| 層1: 業界典型課題 | 「○○業界の典型課題をWeb検索し、課題仮説の裏取り材料を収集」 | 同業種の典型課題 | `outputs/research_layer1.md` |
| 層2: 打ち手事例 | 「課題仮説H番号ごとに、同業・近接業界の改善事例と数字を収集」 | 課題仮説に対応する打ち手事例 | `outputs/research_layer2.md` |
| 層3: 失敗事例 | 「同種の取り組みの失敗パターンと回避策を収集」 | 失敗事例・回避策 | `outputs/research_layer3.md` |

### promptテンプレート（各エージェントに渡す）

```
あなたは「{層の名前}」の調査担当です。

## コンテキスト
- 業種: {案件理解サマリーから}
- 課題仮説: {課題仮説シートのH番号と要点}

## タスク
{層ごとの具体的な調査指示 — references/観点とスキップ基準.md の該当層の記述を転記}

## 制約
- Web検索は5回以内で完了すること
- 各事実に出典リンクを付けること（リンク切れ・伝聞は使わない）
- ベンダー記事の数字は「事例として紹介」レベル。断定しない
- 結果は {出力ファイルパス} に書き出すこと
- メインへの返答は「○件収集。詳細はファイル参照」程度の要約のみ
```

### 統合（手順3）

3本の中間ファイルを読み、メインエージェントが以下の構成でリサーチメモを統合する:
1. 冒頭: 入力と検索実施日
2. 層1の内容（業界典型課題）
3. 層2の内容（打ち手事例 — H番号対応を明示）
4. 層3の内容（失敗事例・回避策）
5. 末尾: 「0-4への示唆」（リサーチが提案のどこに効くかを箇条書き）

統合後、中間ファイル3本は削除する。

## 停止ポイント

- 手順1のスキップ判断（軽い確認）と手順4の軽い確認のみ

## 出力

- `00_前段/03_リサーチメモ_v0.1.md`（冒頭に「入力: 02_課題仮説シート_vX」と検索実施日を明記）

## ルール

- 観点は必ず「クライアントの業種×課題仮説」で組む。汎用リサーチにしない
- ベンダー記事の数字は「こんな事例がある」レベルで使う。鵜呑みにして断定しない
- **確認OK後、裏取りで確定した事実があれば `project-context.md` に書き戻す**（推測・伝聞は書かない）
