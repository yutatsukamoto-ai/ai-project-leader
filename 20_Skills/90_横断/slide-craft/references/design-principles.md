# 伝わるスライドのデザイン原則

> 出典：「伝わる」スライドの創り方（しょこら氏）をベースに体系化。

## 1. メッセージをクリアにする

「伝わらない」スライドの最大の原因はデザインではなく、**作っている本人ですらメッセージが曖昧**なこと。クリアでない考えはクリアに伝わらない。

- **1スライド = 1メッセージ**。一番伝えたい、たった1つのこと。2つあるなら2枚に分ける。
- **メッセージとは「主張」**。「○○の3選」は羅列でメッセージではない。「3つを達成するには△△すべき」のように主張があって初めてメッセージ。箇条書きが刺さらないのは主張がないから。
- **作り方＝書き散らかし → 削ぎ落とし**。
  - 書き散らかし：言いたいことを質を考えず紙に出し尽くす。繰り返し出てくる単語＝メッセージの核。
  - 削ぎ落とし：伝わるか伝わらないかのギリギリまで削る。削るほど残ったものが強くなる。多くの場合「残しすぎ」。
- **注目は取れない前提**で書く。受け手にとってスライドは数ある情報の1つ（One of them）。一目で意味が分からなければ言っていないのと同じ。言葉はシンプルに、横文字（プレイフル等）に頼らない。
- 文字を大きくすると自然に取捨選択が起きる。実務で**16pt以下を使わない**、SNS用なら20ptが下限、という枠の決め方が有効。

順番の鉄則：**タイトル・メッセージ・コンテンツのうち、常にメッセージを先に決める**。コンテンツから作るとメッセージとズレ、手戻りする。

## 2. メッセージを表すスライドにする

メッセージが決まったら「一目でそれが伝わるか」を確認する。

1. **一番言いたいことを表しているか**：コンテンツはメッセージを表すための手段。順番が逆（先にコンテンツ）になっていないか。
2. **一目で大事な箇所に目が行くか**：第一印象は3秒で決まる。3秒で何を言っているか分からなければ受け手に厳しい。メッセージ箇所を目立たせ、真っ先に視線が行くようにする。最も簡単な方法は**赤枠太線で囲う**こと（不格好でも機能する）。そこさえ見れば後はオマケ、というくらい優先順位を付ける。
3. **情報量は過不足ないか**：たいてい多すぎる（Busy／うるさい）。正確さと分かりやすさはトレードオフ。「相手にどこまで分かってほしいか」を決めて詳しさを変える。メッセージに不要な情報は必ず削れる。

## 3. 強調の階層（アクセント設計）

主役を1点に絞る（§2）とき、「どの程度目立たせるか」に6段階の序列がある。1枚のスライドでは上位の段階をメッセージ箇所に限定し、残りは下位に落とす。

| レベル | 手法 | 使いどころ |
|-------|------|----------|
| **Lv6（最強）** | 塗りつぶし背景＋白抜き文字 | ページの核心メッセージ。1枚に1箇所まで |
| **Lv5** | 薄い背景色＋色テキスト | 補助的な強調。Lv6と組み合わせて差をつける |
| **Lv4** | 枠線のみ＋色テキスト | 注意喚起の囲い。Lv5より控えめ |
| **Lv3** | 色テキスト（枠なし） | キーワード・数値のハイライト |
| **Lv2** | 太字（色なし） | 小見出し・ラベル |
| **Lv1（最弱）** | 通常テキスト | 本文・説明文 |

**使い方のルール:**
- 1枚のスライドで使う強調レベルは**最大3段階**。全段階を使うと階層が伝わらない
- メッセージ箇所にはLv5以上を使い、それ以外はLv3以下に抑える
- 白黒先行（横断ガイドライン §9）で構造を確認してから強調を載せる順番は変わらない
- 70:25:5の「5%＝アクセント」はこの序列のLv5〜Lv6に対応する。アクセント色を使える面積が小さいからこそ、段階を分けて使い分ける

> 出典: うちた氏「強調の強さの序列」を6段階に再構成。

## 4. 視線の動きをデザインする

受け手の視線は**上から下、左から右**へ動く。スライドは左上から始まり右下へ進む設計にする。

- **良い事象・結論は右**に置く（視線が最後に行き着くのが右下だから）。比較型では良い方を右に。
- **ハイライトした箇所に直接目が行き、その周辺しか見なくなる**。だから大切な説明はハイライトの周りに固める。離れた場所に置くと見てもらえない。
- **田の字マトリクスは右上が最高の事象**になるよう設計する（一般的なお作法に乗ると伝わりやすい）。
- 例外：**降順の棒グラフは左が高く右に行くほど低く**なる（慣習）。右に行くと小さくなる＝先細りの印象なので、コストや課題など「小さくなると良い話題」に使う。逆に業績の伸びは右に向かって大きくなるグラフにする。

## 5. 配色（黄金の配色比 70:25:5）

色は最も強く、最もセンスを要する要素。センスなしで決めるには黄金比を使う。

- **70 : 25 : 5 = ベース : メイン : アクセント**。
  - ベース（70%）と メイン（25%）は**目立ってはいけない**。背景・大半の要素。
  - アクセント（5%）だけを目立たせる。「ここを見て」の1点に使う。
- **多色は視線を散らす**。色はここぞの箇所にだけ使う。
- ベースとメインを同じ色相にして**透明度（明度）で差をつける**と、選び直し不要で楽。トーンも揃う。
- アクセントは色相環で反対側の**補色**を使うと目立つ。
- 色の一般的なイメージ（表現したい印象から選ぶ）：

  | 色 | イメージ |
  |----|--------|
  | 赤 | 情熱 |
  | オレンジ | 社交 |
  | 黄 | 明るさ |
  | 緑 | 安心 |
  | 青 | 知性 |
  | グレー | 落ち着き |
  | 黒 | 威厳 |

- コーポレートカラーがあればベースに使う。迷ったら**紺かグレーが鉄板**（無難で、赤いハイライトが映える）。黒は強すぎるのでベース/メインにはグレーが扱いやすい。
- **純黒（`000000`）と純白（`FFFFFF`）は使わない**。純黒はコントラストが強すぎて目が疲れる。テキストには`1A1A1A`〜`333333`のダークグレーを、背景には`FAFAFA`〜`F8F8F8`のオフホワイトを使う。自然界に完全な黒・白はほぼ存在せず、わずかに外すだけで柔らかく自然な印象になる。
- **白黒印刷でも区別が付くか**を必ず確認。色だけで分類すると色が消えた瞬間に伝わらない。明度差やラベルでも区別する。

### 5a. アクセント1色族の鉄則

70:25:5を守っていても、アクセント色が複数あると統一感が崩壊する。デッキ全体で**アクセントは1色族（本体＋淡色版の2段階）**に限定する。

- **並列アイテムは同色**: 01/02/03のバッジ、3カラムの左バー、フローの等価ステップなど、対等な項目に異なる色を振らない。位置で区別は十分。
- **色で差をつけるのは意味の差があるときだけ**: Before/After、Go/No-Goは「アクセント色 vs 無彩色（グレー）」の2値で表現。第3の色（緑・金など）を持ち込まない。
- **あしらい（バッジ・コールアウト帯・矢印コネクタ）はアクセント色で統一**: 装飾要素ごとに別の色を使うと散る。全てアクセント色族に揃える。

詳細は横断ガイドライン「スライド生成の原則 §9a」を参照。

## 6. 整列（揃える）

綺麗に見えるスライドの条件は「揃っている」こと。整列はセンス不要で誰でもできる、最もコスパの高い改善。

- 図形・色・テイストが揃うと「秩序・安心」、揃わないと「不協和」の印象。
- **揃っている/いないこと自体がメッセージ**になる。間隔を変えるとグループ分けや仲間はずれを表現できる。逆に**意図しないズレは「中身が整っていない」というノイズ**を出す。
- **縦軸・横軸を意識して要素を揃える**。スライドを「表（縦軸×横軸）」として捉えると配置ルールが生まれ、伝わりやすくなる。
- PowerPointの「配置（整列）」機能やスライドマスターを使い、マウスの目分量で置かない。タイトル・メッセージ・コンテンツの枠をマスターで固定するだけで統一感が出る。

## 7. 細部とよくある失敗

- デザイン（装飾）は**最後の締め**。重要度はメッセージ・構造より低いが、手間がかかっているかは一目で伝わってしまうので軽視はしない。
- フロー図は**一方向**（上→下、左→右）。紙面都合で逆流させると混乱を呼ぶ。
- 表はExcelをそのまま貼らない（Excelは考える道具で見せる道具ではない）。
- グラフは「伝えたいこと」に合わせて簡略化する。期間や推移が不要なら棒グラフで十分なことが多い。

## 8. AI感を消す（pptxgenjs生成の急所）

AIでスライドを生成すると「テンプレ臭」が出やすい。プロのIR資料（資生堂・DeNA等）にはない装飾パターンが原因。以下を避ける。

### 避けるべきパターン

- **丸バッジ＋カード枠＋区切り線の3点セット**：最大のAI感の原因。プロの資料にこの組み合わせはほぼない。番号を見せたいなら丸バッジではなく「大きな薄色テキスト（28pt, 背景的な淡色）」で表現する。カード枠は原則使わず、左アクセントバー（幅0.05in）で区切る。
- **装飾過多**：影・グラデ・角丸・枠線を重ねると「テンプレを埋めた」印象になる。プロの資料は装飾が少ない。構造（配置・余白・フォントサイズ）だけで見せる。
- **意図の見えない配置**：要素が画面の一部に偏り、残りが空白になる。すべてのスペースに「なぜそこに置いたか」の理由が要る。

### 代わりに使うパターン

- **3カラム並列**：左アクセントバー（0.05in幅）＋ 大きな薄色番号（01/02/03, 28pt, accentLight色）＋ タイトル（太字）＋ 本文。カード枠なし。
- **フロー図**：`startX = (10 - totalW) / 2` で全幅中央配置。箱は内容に合わせて十分な幅を取る。矢印は箱と箱の間に等幅で。
- **テーブル**：`colW` の合計を `w` に**厳密一致**させる（ずれると左寄りに見える）。凡例テキストは `tableY + rowH × 行数 + 余白` で座標を正確に計算し、テーブルに被らないようにする。

### 判断基準

「プロが作ったように見えるか」の簡易テスト：スライドから丸バッジ・カード枠・区切り線をすべて消してみて、情報の構造が崩れないなら、それらは不要な装飾。

## 9. レビュー用チェックリスト

既存スライドを診断するときの観点（各ページごと）。横断ガイドライン §12（提出前レビュー5項目）と合わせて使う。

### メッセージ・構造

- [ ] このページの「たった1つのメッセージ（主張）」が言えるか。羅列になっていないか。
- [ ] メッセージとコンテンツが一致しているか。
- [ ] **構造型**（順番/対比/仕分け/問題→打ち手/原因→結果/結論先出し）と実際のレイアウトが合っているか。
- [ ] **見出しが30字以内で言い切れているか**。「〜について」で止まっていないか。主張が含まれているか。

### 視線・配置

- [ ] 3秒で「どこを見ればいいか」が分かるか。強調が機能しているか。
- [ ] 結論・良い事象が右側にあるか。フローは一方向か。
- [ ] **2本の軸で情報が並んでいるか**。軸が3つ以上になっていないか。
- [ ] **同じ言葉が繰り返されていないか**。共通部分を軸のラベルに吸収できないか。

### 情報量・仕上げ

- [ ] 情報量は多すぎないか。削れる情報はないか。
- [ ] **強調の階層が3段階以内に収まっているか**（§3）。Lv5以上がメッセージ箇所に限定されているか。
- [ ] **主役が1点に絞られ、脇役のトーンが落ちているか**。全要素が同じ目立ち方をしていないか。
- [ ] **囲み線でなく余白でグループ化できないか**。不要な枠線がノイズになっていないか。
- [ ] **注釈が欄外に逃がされているか**。本文エリアに混ざっていないか。

### 配色・実装

- [ ] 色は70:25:5に収まっているか。多色で散っていないか。アクセントは1点か。
- [ ] **白黒でも構造が成立するか**（§9）。色を消しても何を言っているか分かるか。
- [ ] 要素は縦軸・横軸で揃っているか。意図しないズレはないか。
- [ ] 丸バッジ＋カード枠＋区切り線の3点セットを使っていないか（§8参照）。
- [ ] テーブルの`colW`合計がテーブル幅`w`に一致しているか。
- [ ] フロー図やカラムの配置が`(10 - totalW) / 2`で中央に来ているか。
- [ ] 凡例・注釈テキストがテーブルや図に被っていないか（座標を計算で確認）。
- [ ] **全テキストが親コンテナ内に収まっているか**。`textX+textW ≤ containerX+containerW` かつ `textY+textH ≤ containerY+containerH` を検算。日本語全角1文字≈fontSize×0.018inで見積もる。

診断は「良い点 → 直すと効く点（優先度順）→ 具体的な直し方」の順で、ページのメッセージを尊重して伝える。
