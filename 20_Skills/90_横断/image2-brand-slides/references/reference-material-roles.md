# Reference Material Roles

参照資料をそのまま模倣せず、何を学び取るかを分けるためのガイド。

---

## 基本方針

参考資料は「見た目をコピーする素材」ではなく、設計判断を増やす材料として使う。

```text
consulting_reference: 形、論理構造、課題の見せ方、工程整理
business_deck_reference: 色階層、パーツ、あしらい、図表の強調方法
brand_anchor: 主色候補、トーン、避けたい印象
deck_theme: 今回の資料で主役にする世界観
```

参照資料の色、フォント、装飾をそのまま移植しない。抽象化した型だけを `design_system.md` と `deck_structure.md` に入れる。

---

## 外資系コンサル系資料から参考にすること

参考にするのは、青い見た目そのものではなく、情報を整理する型。

- 課題 -> 詳細 -> 解決策の対応方向性 -> 今後の対応
- 論点を上位概念から下位概念へ分けるロジックツリー
- 課題箇所だけを囲み、塗り、注釈で強調する課題強調図
- 前提、制約、選択肢、推奨案をつなぐ提案ロジックツリー
- 入力、プロセス、アウトプット、判断軸を横方向に並べるフロー型
- 方向性と今後対応を分ける比較表/対応表
- 工程の上段にステップ矢印、下段に作業/成果物/判断軸を置く構成

避けること:

- 参照資料の青い配色を固定テンプレートとして使う
- 罫線、表、矢印を増やしすぎて密度過多にする
- 文字が小さすぎるコンサル資料をそのまま真似る
- 「表があるからコンサルっぽい」で終わらせる

---

## Retty型資料から参考にすること

参考にするのは、オレンジそのものではなく、色階層とパーツの文法。

- 近い白背景に、主色を一点集中で効かせる
- 見出し番号、章チップ、短いラベルに主色を使う
- 主色の濃い棒/線/数字で、現在値、推奨案、重要項目を立てる
- 主色の薄い面で、強調セル、囲み、補助背景を作る
- 比較対象、過去値、非選択肢はグレーへ退かせる
- 表やグラフでは、全部を塗らずに読むべき列/棒/セルだけを強調する
- 小さなチップ、細い線、薄い囲み、余白でパーツを整理する

避けること:

- どの案件でもオレンジを使う
- 章番号やチップを大きくしすぎる
- すべての要素を主色で塗る
- パーツだけを真似て、ページの主張や判断軸が弱くなる

---

## 設計順序

1. まず資料テーマと読者の判断目的を決める。
2. 外資系コンサル系の型から、ページごとの `logic_pattern` を選ぶ。
3. Retty型の色階層から、ページごとの `color_role` を決める。
4. ブランドアンカーから主色候補を取り、必要なら暗く/鈍く/薄く翻訳する。
5. 部品は、チップ、薄面、囲み、強調線、グレー退避、下段判断軸などから少数だけ選ぶ。
6. 参照元の色やテンプレートに似すぎていないかを検査する。

---

## 標準の考え方

標準は「深緑の見た目」ではなく、以下の設計プロセスである。

```text
theme_translation -> logic_pattern -> color_hierarchy -> component_language -> prompt
```

特定の案件で深緑が合う場合は使ってよい。ただし、深緑は固定デフォルトではなく、テーマに合う主色候補の一つとして扱う。
