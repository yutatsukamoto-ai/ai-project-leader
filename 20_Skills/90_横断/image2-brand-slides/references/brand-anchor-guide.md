# Brand Anchor Guide

`image2-brand-slides` でブランドアンカーを決めるためのガイド。

目的は、テンプレート模倣やWebサイト再現ではなく、現在の入力を資料テーマへ翻訳し、再利用可能な `design_system.md` を作ること。

---

## ブランドアンカーの優先順位

1. ロゴ画像
2. Style Markdown / 社内デザインルール
3. 公式サイトURL
4. 既存資料/PDF/スクリーンショット
5. ユーザー指定の雰囲気・業種・用途
6. 壁打ちによる暫定ブランド指定

ロゴがない場合でも止めない。ただし、ロゴを捏造しない。

ブランドアンカーは「そのまま似せる対象」ではなく、資料の目的・読者・意思決定テーマに合う世界観へ翻訳する材料として扱う。ユーザーが明示しない限り、会社サイト風、ロゴ主役、スクリーンショット主役にはしない。

---

## `input_manifest.md` に残すこと

必須:

- 入力種別
- 参照URLまたはファイル名
- 取得日・確認日
- 抽出した色（可能ならHEX）
- 抽出したトーン・雰囲気
- 推測と事実の切り分け
- 参照しなかったもの

例:

```md
# input_manifest

## 入力

- 原稿: 入力_研修メモ.md
- ブランドアンカー: 公式サイト
- 参照URL: https://example.com/
- 確認日: 2026-06-18

## 抽出した事実

- メインカラー: #123456
- 背景: 白基調
- 写真傾向: 人物よりプロダクト画面中心

## 推測

- トーン: 技術的、落ち着いた、B2B寄り

## 参照しなかったもの

- 過去の生成画像
- 無関係なテンプレート
```

## ロゴあり

見るもの:

- 主色
- アクセント色
- ロゴの縦横比
- 角の丸さ/鋭さ
- 線の太さ
- 企業感、技術感、親しみ、プレミアム感など

ルール:

- タイトルスライドのロゴは小さく控えめにする。
- ロゴを主役にしない。ただしユーザーがロゴ前面を明示した場合は別。
- 通常スライドのロゴ位置は右上で統一し、後処理で揃える。
- 明るすぎるロゴ色は、資料テーマに合う深い/鈍い色へ翻訳して使う。

---

## Style Markdownあり

Style Markdownは、ロゴと同等以上に強いブランドアンカーとして扱う。

見るもの:

- 基本方針
- カラー
- レイアウト
- タイポグラフィ
- 図解傾向
- 避けたい表現

ルール:

- `design_system.md` の一次入力にする。
- ロゴや公式サイトと矛盾する場合は、勝手に統合せず差分を記録する。
- 古い・抽象的・不足している場合は、暫定解釈として扱う。
- 「避けたい表現」はImage2プロンプトのnegative constraintsへ反映する。

---

## 公式サイトあり

ユーザーから明示されたURLを優先する。必要ならWeb閲覧して確認する。

見るもの:

- ファーストビューの色
- 背景色
- ボタン/リンクのアクセント色
- 余白
- 写真/図版の使い方
- 見出しの言葉遣い
- B2B/B2C、技術/人間味、堅め/軽め

注意:

- 公式サイトから見える現在の情報だけを使う。
- 古い記憶や推測で補わない。
- 取得日を `input_manifest.md` に残す。
- 公式サイトの画面構成、ファーストビュー、写真、ボタン色をそのまま再現しない。
- サイトの主色が明るい場合は、提案テーマに合わせて深色・低彩度・少量使いへ翻訳する。

---

## 既存資料/PDF/スクリーンショットあり

見るもの:

- 余白
- 見出し位置
- 色数
- 表/グラフの密度
- 写真やアイコンの使い方
- 社内資料らしさ、顧客向け資料らしさ

注意:

- 既存資料を丸写ししない。
- 「型」ではなく、色・余白・情報密度・言葉遣いを抽出する。
- 実案件データや顧客固有情報を配布物へ混ぜない。
- スクリーンショットを大きなビジュアル素材として主役化しない。必要な場合も、参照元として薄く扱う。

---

## 雰囲気指定のみ

ユーザー指定の言葉から暫定デザインシステムを作る。

例:

- シンプル
- コンサルっぽい
- 先進的
- 親しみやすい
- 役員向け
- 現場向け
- 研修資料
- editorial proposal style
- 静かな提案資料
- 生成り、主色の濃淡、線画、余白多め
- 外資系コンサル資料のような論理構造
- Retty型の色階層やパーツ感

不明確な場合は3問だけ聞く。

1. 誰に見せる資料か
2. どんな印象にしたいか
3. 避けたい印象は何か

---

## ロゴなし

ルール:

- ロゴを捏造しない。
- 通常スライドの右上は空けるか、ユーザー指定の短いデッキマークのみ使う。
- ヘッダー補正は小タイトルと短い下線だけにする。
- no-logoであることを `input_manifest.md` に明記する。

---

## 安全なデフォルト

ブランド情報が薄いときの初期値:

- 生成り/オフホワイト背景
- 資料テーマに合う主色を1色だけ選ぶ。深緑は候補の一つであり固定値ではない
- 主色の濃淡で、見出し、薄い面、線、チップ、強調棒を分ける
- 濃いグレー本文と、非強調要素のグレー退避
- アクセントは推奨、注意、結論、差分だけに限定する
- 広い余白
- 低装飾
- 薄い罫線
- Noto Sans JP風の細い日本語サンセリフ
- 1スライド1メッセージ
- 2〜3グループまで
- thin line icons、意味のある抽象図、必要な場合だけ下部メッセージボックス
- ロジックツリー、課題強調図、対応表、入力/プロセス/アウトプット型フローを候補に入れる

避けるもの:

- ブランド模倣
- 公式サイト風のWebデザイン
- ロゴ/キャラクター/スクリーンショットの主役化
- 明るすぎるブランドカラーの全面使用
- 深緑/セージ、Retty風オレンジ、外資系資料の青い見た目への固定化
- 汎用カードUIや3カラムカードの乱用
- 角丸 + 薄影のAI資料っぽさ
- 装飾的な背景
- 派手なグラデーション
- 意味のない写真素材
- 多色のカード
- 重い太字だらけの文字
- ダッシュボード風の詰め込み
