## 修正一覧

### 1. [対策D] check_selection_guide_coverage を verify() に追加

`_tools/_lib.sh` に関数 `check_selection_guide_coverage` を新設し、`_tools/build.sh` の `verify()` 内で呼び出す。

目的: 成果物マップに載っているSkillが成果物選定ガイドに漏れていないかを機械検知する。

ロジック:
1. `20_Skills/成果物マップ.md` のフェーズ別テーブルから、Skill名（ハイフン区切り英小文字）を抽出する。除外: chainの親Skill（`zendan-chain`, `tachiage-chain` 等 `-chain` で終わるもの）と未整備候補セクション内のもの
2. `40_Stock/横断ガイドライン/成果物選定ガイド.md` のランク表（A節）からSkill名を抽出する。テーラリング発動型セクションの `Skill` 列も含める
3. 成果物マップにあって選定ガイドにないSkill名を警告: `❌ 選定ガイド未収録: {Skill名}`
4. 選定ガイドにあって成果物マップにないSkill名も警告（孤児エントリ）: `❌ 成果物マップに不在: {Skill名}`

verify() への組み込み位置: `check_skill_name_ghosts` の直後に以下を追加:

```bash
  echo "=== 選定ガイドと成果物マップの整合 ==="
  check_selection_guide_coverage "$ROOT" || problems=$((problems+1))
```

注意:
- 成果物マップの「未整備領域（将来Skill化候補）」セクション内のものは対象外（まだSkillが存在しない）
- 選定ガイドの「未整備領域」セクションも同様に対象外
- 成果物マップの「オーケストレーター」セクションのchainも対象外（選定ガイドは成果物単位）
- 成果物選定ガイドのテーブルにはSkill名が明記されていない行もある（本体チェーンの「課題管理表」等は成果物名のみ）。その場合は成果物名をキーにして `20_Skills/` 配下のディレクトリ名と照合する必要はない — Skill名が明記されている行だけを比較対象にする
- 誤検知が多い場合は `_tools/selection-guide-allowlist.txt`（1行1語）で除外する設計にする

### 2. 横断GL READMEの索引漏れチェックを verify() に追加

`_tools/_lib.sh` に関数 `check_gl_readme_index` を新設し、`_tools/build.sh` の `verify()` 内で呼び出す。

目的: `40_Stock/横断ガイドライン/` に .md ファイルを追加したのにREADME索引表に載せ忘れるのを防ぐ。

ロジック:
1. `40_Stock/横断ガイドライン/` 配下の `*.md` ファイル一覧を取得（`README.md` 自体は除く）
2. `40_Stock/横断ガイドライン/README.md` の索引テーブルの「正典ファイル」列からファイル名を抽出
3. ディレクトリにあってREADMEに載っていないファイルを警告: `❌ 横断GL README索引漏れ: {ファイル名}`
4. READMEに載っているがディレクトリに存在しないファイルも警告: `❌ 横断GL 幽霊索引: {ファイル名}`

verify() への組み込み位置: `check_selection_guide_coverage` の直後に以下を追加:

```bash
  echo "=== 横断GL README索引の整合 ==="
  check_gl_readme_index "$ROOT" || problems=$((problems+1))
```

注意:
- README.md内のテーブル行はバッククォートでファイル名を囲んでいる（例: `` `文章作法.md` ``）。パース時にバッククォートを除去すること
- `成果物Skill共通ひな形.md` は横断GLフォルダ内に存在するが、READMEの索引表にも載っている。漏れなく照合すること
