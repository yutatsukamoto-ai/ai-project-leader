# Skillチェーン構築プロジェクト 引き継ぎ文書

これは別チャットでClaudeと議論して固めた設計方針の引き継ぎです。このプロジェクトでは、以下の前提を共有済みのものとして壁打ち・作業を進めてください。

---

## 1. やりたいこと(ゴール)

私は「AIプロジェクトリーダー」として、生成AIを活用してコンサルティングプロジェクトを推進していきたい。
そのために、**PM業務の各成果物(業務定義シート、業務の地図、業務フロー図、課題一覧、As-Is/To-Be、WBS、提案書スライドなど)を生成するSkill群と、それらを束ねるオーケストレーターSkillを構築・運用する**。

対象プラットフォームは Claude Code / Claude Cowork の両方(将来的にCopilot等へのポータビリティも考慮)。

## 2. 合意済みの設計方針

### 基本アーキテクチャ:「1成果物=1Skill + 薄いオーケストレーター」

- **子Skill**: 成果物1つにつきSkillを1つ作る。各Skillが品質基準・書き方の知識を持つ
- **親Skill(オーケストレーター)**: 「順番・成果物の受け渡し・どこで人間に確認を挟むか」**だけ**を持つ10〜20行の薄いSKILL.md。中身の知識(分類方法、デザインルール等)は絶対に親に書かない
  - 理由: 二重管理の防止 / 子Skill単体利用の維持 / チェーンの組み替え自由度

### 一括生成と個別生成の使い分け基準

**「前の成果物が間違っていたとき、後ろを直すコストがどれだけ高いか」**で判断する。

- 同じ入力から作れて修正コストが低い成果物(例: 業務定義シート+業務の地図)→ 1ステップで一気に生成してよい
- 前工程の正しさに品質が依存し、作り直しが重い成果物(例: 業務フロー図、WBS)→ その手前に必ず人間の確認ポイント(停止)を置く

### チェーン設計の思想

- 前のSkillの出力が次のSkillの入力になる「コンボ攻撃」型(PAC思考ベース)
- 1段ごとに人間がレビューして磨いてから次に渡すことで、粗さの伝播を防ぐ

## 3. 運用に必要なアウトプット(最小3点セット)

```
{chain-name}/                    ← 親Skill(オーケストレーター)
├── SKILL.md                     ← ①実行手順書(順番+停止ポイントのみ)
└── templates/
    └── project-context.md      ← ②共有コンテキスト雛形

{child-skill-a}/
└── SKILL.md                     ← 子Skill(成果物ごと)
{child-skill-b}/
└── SKILL.md
```

1. **親SKILL.md**: 手順(どのSkillをどの順で呼ぶか)、【必ず停止】する確認ポイント、「各ステップの入力は前ステップのoutputs/ファイルのみ」「確認ポイントを勝手にスキップしない」のルール
2. **project-context.md**: クライアント名・目的・制約等を一元管理する共有コンテキスト。全Skillがこれを読む(前提を毎回口頭で伝え直さないため)
3. **outputs/の命名規則**: `outputs/01_課題一覧.md` のような番号付き命名。受け渡しの確実性と、途中再開(セーブポイント)を担保する

※ `run-log.md`(実行履歴)は運用が回り始めてから必要に応じて追加。最初から作り込まない。

## 4. 起動・運用イメージ(共有済み)

- Claude Code: `/chain-name` で明示起動、または自然文からdescription経由で自動起動
- 子Skillの「起動」= 同じClaudeが子SKILL.mdを読み込んでその指示に従うこと(会話・文脈は1本のまま)
- descriptionにトリガーワードを充実させる(自動起動の精度のため)
- チェーン全体を回したいときは親の名前で指名する(子Skillだけ直接走るのを防ぐ)
- Skill完成時は .skillファイル(ZIP)としてパッケージし、ダウンロード可能な形で渡すこと

## 5. このプロジェクトでの進め方

アウトプットイメージのリサーチはすでにある程度完了している。ここからは:

1. 対象とする成果物チェーンの確定(どの成果物を、どの順で、どこに確認ポイントを置くか)
2. 子Skillの個別設計・作成(1つずつ品質を確認しながら)
3. 親Skill(オーケストレーター)の作成
4. 実案件での試運転 → 改善

の流れで、壁打ちしながら進めたい。多段階のタスクは勝手に進めず、各ステップで確認を取りながら進めてください。
