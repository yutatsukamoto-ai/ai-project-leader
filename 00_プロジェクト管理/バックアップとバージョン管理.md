# バックアップとバージョン管理

最終更新: 2026-06-15
狙い: 手作りで積み上げた構造・Skill・正典を「消失・誤編集で巻き戻せない」リスクから守る（致命的＝不可逆ミスの防止）。

## 1. いま効いている保険：フルスナップショット

- 場所: `_backups/AIPL_snapshot_YYYYMMDD_HHMM.tgz`（全部入り＝著作物・実データも含むローカル保管用。配布物ではない）。
- 復元: ターミナルで `tar -xzf _backups/AIPL_snapshot_xxxx.tgz` → フォルダがまるごと展開される。
- 注意: これは「点のバックアップ」。日々の細かい巻き戻しには下の git が要る。
- 推奨: 余裕があれば、この `.tgz` を外付け/クラウドにも1部置くと一段安全（同じディスクの故障に強くなる）。

## 2. バージョン管理（git）は「自分のターミナルで」セットアップする

なぜ手元で: このアシスタントはサンドボックス越しにフォルダを触っており、その環境が git の内部ファイル操作（`.git` 内の作成・削除）をブロックする。実際に試したが `.git` を正しく作れなかった。**あなたのMacのネイティブ環境なら問題なく動く。**

`.gitignore` は用意済み（著作物・実データ・ビルド成果物・バックアップを除外）。やることは下の4行だけ：

```bash
cd ~/AIプロジェクトリーダー
rm -rf .git                       # ← アシスタントが残した不完全な .git を消す（重要）
git init
git add -A && git commit -m "初期コミット: 構造・Skill・ツール一式"
```

### セットアップ後の確認（除外が効いているか）

```bash
git status                        # 八束電工・書籍PDF・画像が出てこなければOK
git ls-files | grep PMBOK         # PMBOK正典mdの2本は追跡されている＝OK
```

## 3. 運用ルール

- **節目でコミット**: Skillを足した／正典を直した／案件を一周した、など区切りで `git add -A && git commit -m "..."`。
- **巻き戻し**: 1ファイルを戻すなら `git restore <ファイル>`、直近コミットの状態を見るなら `git log` / `git show`。
- **push する前に必ず配布除外を再確認**: リモート（GitHub等）に上げる瞬間がいちばんの事故ポイント。`.gitignore` が効いているか＋協和精機など他の実情報案件がないかを D-01（実データ匿名化）と合わせて点検してから push する。ローカルに留める限りは事故らない。
