# パーツ・あしらいパターン辞書

> 出典：『資料作成デザインアイデア図鑑』Ch4（パーツ・あしらいのデザインアイデア）の知見を、pptxgenjs実装向けに再構成。
> スライド生成時、装飾パーツの選択・実装判断に使う。

## 使い方（最初に読む）

パーツ・あしらいは**メッセージを補助する脇役**。主張そのものを変えはしないが、適切に使えば視認性・印象が上がる。

**原則:**
- あしらいが**目立ちすぎたら逆効果**。構造（配置・余白・サイズ）が先、あしらいは最後
- 1スライドで使うあしらいは**2種類まで**。組み合わせすぎるとごちゃつく
- design-principles.md §8（AI感を消す）と矛盾するパターンは使わない

---

## 1. ナンバリング

番号をただ置くのでなく、あしらいで区別をつける。

| パターン | 説明 | pptxgenjs実装のヒント |
|---------|------|---------------------|
| 大きな薄色番号 | 28pt以上、accentLight色で背景的に配置。**推奨（AI感が出ない）** | `addText('01', {fontSize:28, color:accentLight, bold:true})` |
| 左アクセントバー＋番号 | 細い縦線（w=0.05in）の右に番号。3カラム並列と相性◎ | `addShape('rect', {w:0.05, fill:accent})` + テキスト |
| ゼロ埋め（01, 02...） | 桁揃えで視覚的に統一感が出る | 文字列で `'01'` |
| #付き（#1, #2...） | カジュアルな印象 | テキストのみ |
| Vol. / No. / Ver. 付き | 正式感・シリーズ感を出す | テキストのみ |
| 下線付き番号 | 番号の下に短い線を引く。シンプルで汎用的 | `addShape('line')` を番号下に配置 |

**避けるパターン:** 丸バッジ（`ellipse`＋中央番号）はAI感の最大原因。design-principles.md §8参照。

---

## 2. 矢印

矢印は**補助パーツ**。目立ちすぎないように。

| パターン | 印象 | 実装 |
|---------|------|------|
| 線＋三角形 | スタンダード・きれい | `line` + 小さな `triangle` |
| くの字（シェブロン） | モダン・軽い | `chevron` shape |
| 三角形のみ | ミニマル | `triangle` shape, 小さめ |
| 3つの角丸四角形を並べる | 柔らかい・ステップ感 | `roundRect` ×3, サイズ漸増 |
| 2つの平行四辺形 | ダイナミック・動き | `parallelogram` ×2 |

**原則:** 矢印はあくまで方向を示す補助。太すぎ・大きすぎは禁物。色はメインカラー（25%）以下に。

---

## 3. 吹き出し

ポップ・親近感を出したいときに。ビジネス資料では使いすぎ注意。

| パターン | 印象 |
|---------|------|
| 角丸四角形＋小三角形 | オーソドックス。pptxgenjsでは四角形＋三角形を重ねて作る |
| 線だけの吹き出し（塗りなし） | 軽い・抜け感。線の四角形＋線の三角形 |
| 同じ形をずらして重ねる | 影のような奥行き感。同色の濃淡で |
| カクカクした吹き出し | ポップ・勢い。頂点を編集した多角形 |

---

## 4. あしらい（テキスト装飾）

テキストブロックにワンポイント加える手法。

### 線系（軽め・ビジネス向き）

| パターン | 説明 |
|---------|------|
| 線で挟む | テキストの上下に水平線。セクション区切りにも |
| 右と下に線 | L字型。テキストブロックの右端と下端に細線 |
| テキストに重ねて図形を配置 | 薄い色の四角形をテキスト背面にずらして配置。奥行き感 |

### 図形系（やや装飾的）

| パターン | 説明 |
|---------|------|
| 図形を背面に敷く | 大きめの薄色rectをテキスト背面に。エリア区分に |
| 同じ図形の塗りと線を重ねる | 塗りrect＋やや大きい線のみrectを重ねる。ワッペン風 |
| 枠に文字を添わせる | 線のrectの辺上にテキストを配置 |
| 1つの角を空ける | 四角の囲みで一角だけ線を切る。抜け感 |

### 使い分けの目安

- ビジネス・フォーマル → 線系のみ
- セミフォーマル → 線系＋薄い背景図形
- カジュアル・ポップ → 図形系OK（ただし1スライド1種類まで）

## 5. 日付表記

日付のフォーマットで印象が変わる。

| パターン | 例 | 印象 |
|---------|---|------|
| ドット区切り | 2026.06.16 | モダン・スタイリッシュ |
| スラッシュ区切り | 2026/06/16 | スタンダード |
| 月の数字を小さく | 6月 **16** | 日にちを強調 |
| 縦方向に並べる | 2026（改行）06.16 | デザイン性・表紙向き |
| 丸で囲む | ○の中に月/日 | カレンダー風・イベント感 |

## 6. 写真のあしらい

写真をそのまま貼るだけでなく、あしらいでオリジナリティを出す。

| パターン | 実装のポイント |
|---------|--------------|
| 細い線で囲む | 画像より少し大きいrect（line only）を背面に |
| 図形をずらして重ねる | 画像と同サイズの塗りrectを斜め下にずらして背面配置 |
| 角に図形を配置 | 画像の対角2箇所に小さな正方形や三角形 |
| 線をはみ出させて囲む | 画像より大きい枠線を配置。線が画像からはみ出る |
| 下側に長方形をつける | 画像の下にキャプション帯。色付きrectにテキスト |
| 影をつける | pptxgenjsの`shadow`プロパティ。透視投影下が自然 |

**注意:** あしらいが写真より目立つと本末転倒。線は細く（0.5〜1pt）、色は控えめに。

## 7. スライド背景

白が基本だが、変化をつけたいときに。

### 色系

| パターン | 用途 |
|---------|------|
| 薄い灰色（F5F5F5〜EEEEEE） | 白より柔らかい。本文スライドに |
| 薄いグラデーション（白→灰） | 奥行き感。トップ→ボトム方向 |
| 濃いグラデーション | インパクト。表紙・章扉向き |
| 黒背景（1A1A1A〜333333） | シック・高級感。純黒は使わない |
| 薄い色の背景 | ブランドカラーの明度を上げて使う |

### テクスチャ系（画像背景）

| パターン | 印象 |
|---------|------|
| 方眼 | 技術・精密・設計 |
| ドット | 軽やか・モダン |
| ストライプ（細） | スタイリッシュ |
| 紙・ノート | 手作り感・親しみ |
| 大理石・コンクリート | 高級・堅実 |
| 木目 | ナチュラル・温かみ |

**実装:** pptxgenjsでは`slide.background = {path: 'image.png'}` または `{fill: color}`。テクスチャは画像として用意が必要。コード生成ではグラデーション・単色が現実的。

## 8. 数値の見せ方

数値を印象的に見せる手法。KPI・実績スライドで頻出。

| パターン | 説明 | 実装 |
|---------|------|------|
| 単位を数字より小さく | 「**120**億円」→ 120は36pt, 億円は16pt | 別テキストボックスで並べる |
| 小数点以下を小さく | 「**98**.5%」→ 98は36pt, .5%は18pt | 同上 |
| 単位を囲む | 「%」を丸や四角で囲む | shape + テキスト |
| 矢印で補足 | 数値の横に↑↓で増減を示す | 三角形shape（緑=増/赤=減） |
| 優先度で大小差 | 最重要KPIを最大サイズ、他は控えめに | fontSize差で序列を明示 |

**鉄則:** 数値と単位を同じサイズにしない。数値＞単位のサイズ差が視認性の鍵。

## 9. ページ番号

全ページに入る小さなパーツだが、統一感に直結する。

| パターン | 説明 |
|---------|------|
| 線のあしらい | 番号の上や横に短い線 |
| 図形を先頭につける | 小さな正方形や丸を番号の前に |
| 角に吸着 | スライドの角にぴったり寄せる |
| ／で区切る | 「3 / 12」形式。全体感が出る |
| 線で挟む | 番号を上下の短い線で挟む |

**実装:** pptxgenjsでは`slide.slideNumber`プロパティ、またはマスターに直接配置。

---

## 10. アイコン

| ルール | 説明 |
|--------|------|
| 塗り or 線で統一 | 1資料内でアイコンスタイルを混在させない |
| 線を太くしすぎない | つぶれて判別不能になる |
| 図形で囲む | 丸や角丸rectの中にアイコンを配置すると統一感 |
| 図形を背面にずらして重ねる | アイコンの後ろに薄色の丸を少しずらして配置。奥行き感 |

---

## 11. 引き出し線

グラフ・写真・地図から説明を引き出すときに使う。

| パターン | 印象 |
|---------|------|
| 実線 | スタンダード |
| 点線 | 軽い・控えめ |
| 先端に円 | ポイントを明示 |
| 直角に曲げる | 整然・技術的 |
| 斜めに曲げる | 柔らかい |
| 先端に正方形 | モダン |

**原則:** 引き出し線は細く（0.5〜1pt）。色はメインカラーかグレー。

---

## 12. 囲み（フレーム）

情報をグループ化して見せる手法。**余白でグループ化できるなら囲みは不要**（design-principles.md §9参照）。

| パターン | 説明 | 印象 |
|---------|------|------|
| 塗り＋線の四角形 | 薄い背景色＋細い枠線 | スタンダード |
| 四角をずらして重ねる | 同じrectを少しずらして2枚。奥行き感 | モダン |
| テキストを枠の線上に配置 | 枠線の上辺にラベルテキストを重ねる | フォーマル |
| 大かっこで挟む | 左右に「[ ]」形の線 | 軽い・数学的 |
| 影をつける | 薄い影でカードっぽく浮かせる | モダン |
| 角にアクセント | 四角の角にだけ短い線を配置（全辺は引かない） | スタイリッシュ |
| 縦線で挟む | 左右に縦線のみ。上下は開放 | 軽い・引用風 |
| 囲み＋テキスト帯 | 枠の上辺に色帯＋白テキストのラベル | 構造的 |

**注意:**
- カード枠（roundRect＋枠線）の多用はAI感の原因（design-principles.md §8）
- 囲みの線は細く（0.5〜1pt）。太い枠線は圧迫感が出る
- 囲み内テキストはパディングを十分取る（左右0.12〜0.15in、上下0.1in）
